AnimaモデルはYoitomoshi Forge Studioで使えるか

CircleStone Labs / Comfy Org の公式Anima、Civitai公式配布、Stable Diffusion WebUI Forge本家、Forge Neo、 ComfyUI、二次情報、Yoitomoshiローカル実装と過去エラー記録を照合した互換性レポート。

結論: 現在のYoitomoshi Forge Studioが同梱している lllyasviel/stable-diffusion-webui-forge 経路では、 公式Anima Base v1.0を通常のStable Diffusion checkpointとしては使わない。 実際に anima-base-v1.0.safetensors 選択で Failed to recognize model type! が出た記録があり、現行コードもAnima系DiTを unsupported_diffusion として隔離する設計になっている。 使うなら、最短はComfyUI経路。Forge風GUIで使うならForge Neoを別runtimeとして検証する必要がある。

1. 公式Animaの構成

  • 配布元は circlestone-labs/Anima。
  • 2B text-to-image model。CircleStone Labs と Comfy Org の協力によるアニメ、イラスト、非写実系コンテンツ向けモデル。
  • 公式の実行先はComfyUI。Hugging Face上では diffusion-single-file と comfyui タグが付いている。

Yoitomoshiローカル実装の証拠

1. 過去の実エラー

docs/FORGE_MODEL_TYPE_ERROR_FIX_2026-05-15.md には、Forgeの選択checkpointが anima-base-v1.0.safetensors になったときに、txt2img APIで Failed to recognize model type! が返ったことが記録されている。 原因は、Forgeが期待するStable Diffusion checkpointキー model.diffusion_model.* / conditioner.* / cond_stage_model.* ではなく、Animaの net.blocks.* 系キーだったこと。

2. 現行コードの分類

electron/safetensors-inspect.ts はAnima / Flux系のDiTを unsupported_diffusion として分類する。 electron/ipc-handlers.ts のモデル自動振り分けでは、この種別を runtime/forge/webui/models/Unsupported-StableDiffusion へ送る。 これは「Forgeに無理に読ませない」ための再発防止策として妥当。

3. 現在のローカル在庫

ファイル 場所 Yoitomoshi分類 観察結果
lizanima_v10.safetensors runtime/forge/webui/models/Unsupported-StableDiffusion Unsupported header sampleは net.blocks.*。公式Anima系の構造で、現在のForge checkpointとしては読ませない。
dasiwaAnima_luminousLabyrinthV1.safetensors runtime/forge/webui/models/Stable-diffusion Checkpoint header sampleは model.diffusion_model.*。Yoitomoshiモデルライブラリ上はCheckpoint扱い。 ただし、公式Anima Baseがそのまま使える根拠ではなく、Forge向けに変換または調整された派生として別扱いにする。

実運用への推奨

推奨1: 公式AnimaはComfyUI側で扱う

公式Animaを最短で使うなら、Yoitomoshi-Art-GeneratorのForge runtimeではなく、 C:\宵灯工房アート\Yoitomoshi-Art-Generator-ComfyUI またはComfyUI V2経路に置く。 公式ドキュメントと二次情報が同じ方向を指しており、モデル構成もComfyUI前提。

推奨2: Forge Neoは別runtimeで検証する

どうしてもA1111 / Forge風GUIでAnimaを触るなら、runtime/forge を置き換えず、 runtime/forge-neo のような別ディレクトリで検証する。 最低限、/sdapi/v1/options、/sdapi/v1/sd-models、 /sdapi/v1/sd-modules、txt2img、img2img、VAE / Text Encoder選択、 YoitomoshiのModel Library、Preflight、拡張payload、AnimateDiff周辺のスモークが必要。

推奨3: 既存ForgeのStable-diffusion配下に公式Animaを戻さない

anima-base-v1.0.safetensors や lizanima_v10.safetensors のような net.blocks.* 系ファイルをStable-diffusion配下に置くと、モデル一覧には見えても生成時に失敗する可能性が高い。 現行の unsupported_diffusion 分類は維持する。
