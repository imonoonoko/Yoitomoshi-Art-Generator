Character Compositing Workflow

AIキャラを元画像へ追加するための簡単ワークフロー

GitHub/記事調査で得た「LayerDiffuseで透明キャラ」「Inpaint + ControlNetで馴染ませ」 「IP-Adapterで参照」「IC-Lightで照明合わせ」の流れを、まずForge APIだけで使える 置く → マスク → ControlNet/Inpaint準備 のUIとして実装した。

ユーザー操作

  1. img2imgタブで元写真を読み込む。
  2. 「AIキャラ追加」を開き、透明PNGまたはキャラ画像を読み込む。
  3. プレビュー上でキャラ位置をドラッグし、サイズ・回転・左右反転を調整する。
  4. プリセットを選ぶ。夜の写真なら「夜写真」が既定。
  5. 「馴染ませ準備」または「準備して生成」を押す。

内部で行うこと

合成入力を作る

元画像へキャラレイヤーを描画し、暗さ・彩度・ぼかし・影をプリセットで仮適用する。

Inpaint maskを作る

キャラ画像のalphaを白黒mask化し、拡張と境界ぼかしをかけて周辺だけ再描画できるようにする。

ControlNetを自動設定

lineart/canny/softedge系を優先してUnit 1へ設定し、配置の輪郭を生成時に維持する。

調査ソースからの採用判断

今すぐ採用

  • Inpaint + ControlNet: ラフ配置をControlNetで支えながらinpaintする考え方。
  • sd-webui-controlnet: img2img/inpaintとmask対応、Pixel Perfect、preprocessor preview。
  • Replacer: mask + inpaintの一体化UIの参考。

次段階で統合

  • sd-forge-layerdiffuse: 透明キャラ生成とレイヤー合成。
  • IP-Adapter: キャラ参照/顔・雰囲気保持。
  • IC-Light: 背景に合わせた照明調整。
  • Inpaint Anything: SAM由来のmask作成。
  • GLIGEN / Add-it: bbox/object insertion系の研究枠。

次の改善

  • 実写真と透明PNGでElectron目視QAを行い、スクリーンショットを保存する。
  • 生成候補のBefore / 仮配置 / Afterを履歴またはWorkspaceへまとめて保存する。
  • LayerDiffuseで透明キャラを直接生成し、そのままAIキャラ追加へ送るボタンを追加する。
  • IP-Adapter / reference_onlyを「同じキャラを保つ」スロットとして追加する。
  • IC-Lightまたは簡易色調推定で、暗さ・色温度・粒状感の自動合わせを強化する。
