Animadex API 連携 要件・実装設計

Anima向けキャラクター・絵師検索エンジン Animadex を、Yoitomoshi Forge Studio の Prompt Helper、 Prompt Composer、Model Library、CivitAI導線へ接続するための要件定義と実装方針。 $define-requirements の軽量フローで、目的、範囲、非目標、受け入れ条件、リスク、実装手順を整理した。

結論: まずは「Animadexを生成エンジンとして扱う」のではなく、 外部知識ソースから prompt / character trigger / common tags / CivitAI LoRA候補を引く補助APIとして統合するのが最も相性がよい。 現行Yoitomoshi Forgeは公式Anima Baseを通常checkpointとして扱わない設計なので、 生成実行は既存Forgeのまま、Animadex由来の候補は「Animaでの知識可視化から得た制作ヒント」として表示する。 将来のForge NeoまたはComfyUI経路では、同じクライアントをAnima専用Preflightに昇格できる。

1. 前提と補正

ユーザー向け紹介文実装判断での扱い
「3.4万人のキャラクターと1.5万人の絵師タグを完全網羅」 実測APIでは /api/characters/facets が total=34301、 /api/artists/facets が total=15879 を返した。 ただし「完全網羅」ではなく、Danbooru由来データと生成済みサンプル範囲の検索インデックスとして扱う。
「キャラ崩れによる無駄な生成コストを削減」 正しい。特に trigger、作品名、共通タグ、サンプル画像を事前確認できるため、YoitomoshiのPrompt Helperに向く。 ただしサンプルはone-shot生成なので、正確性保証ではなく「候補の強さ」の目安にする。
「CivitAIとの自動連携」 Animadex側はキャラクターに紐づくLoRAを返す。 Yoitomoshi側では既存のCivitAI検索・download・source metadata保存へ橋渡しし、直接ダウンロード前にCivitAI詳細を再確認する。
「VRAM 8GB程度で動作」 ここはレポート上では断言しない。 Anima公式はComfyUI native supportと推奨設定を示すが、8GB実用性は解像度、sampler、quant/low bits、Forge Neo/ComfyUI構成で変わる。

2. Animadex側の確認結果

項目確認内容Yoitomoshiで使う意味
公開サイト animadex.net は、34,000件超のcharactersと15,000件超のartistsを検索できると説明している。 ユーザーがブラウザで確認できる外部参照として、候補カードから開く価値がある。
公開API 現行実装は /api/characters/search、/api/artists/search、 /api/copyrights/search、/api/<mode>/facets、 /api/<mode>/facet/<name> を持つ。 Live GETでは Cache-Control: public, max-age=60 を確認。 main processからfetchする薄いclientで十分。Rendererから直接叩かずIPCで型とcacheを挟む。
GitHub zetaneko/AnimaDex はMITで公開済み。確認commitは af0cee9。 READMEはFlask backend、vanilla JS frontend、SQLite storage、Windows用install/runを示す。 API変更時は公開実装を見て追従できる。将来dataset公開後はローカルmirror/importも現実的。
CivitAI LoRA同期 AnimaDexはCivitAI APIで types=LORA、tag=character、 baseModels=Anima を取得し、キャラslugへfuzzy matchしている。 Yoitomoshi側はAnimadexの一致結果をそのまま信用しすぎず、CivitAI model idで詳細を再取得してdownload候補化する。

GET https://animadex.net/api/characters/search?q=hatsune%20miku&page=1&sort=count
GET https://animadex.net/api/characters/search?q=asuka&page=1&sort=count&loras=1

mode主要query主な戻り値
characters q, page, sort=count|az|random, facets, loras=1 slug, name, copyright_name, trigger, tags, count, thumb_url, img_url, loras[]
artists q, page, sort=count|az|score|random, facets slug, name, trigger, score, thumb_url, img_url
copyrights q, page, sort=count|az|random slug, name, count, thumb_url

3. 目的

Yoitomoshi Forge Studio上で、キャラクター名、作品名、見た目、絵師名から 使えるprompt候補、Anima用trigger、共通タグ、関連LoRA、サンプル画像を短時間で引けるようにする。 既存のPrompt Helperが自然文メモからルールベースに候補を出すのに対し、Animadex連携は 「Animaが実際にどのタグをどれくらい持っていそうか」を外部インデックスとして補う。

4. スコープ

優先度範囲説明
MVP Animadex検索IPC electron/animadex-api.ts を追加し、character / artist / copyright検索をmain processで実行。 timeout、schema normalize、query cache、stale fallbackを実装する。
MVP Prompt Helper連携 PromptHelperPanel に「Animadexで探す」導線を追加。 結果カードから trigger、tags、artist tokenをPromptまたはPrompt Composer slotへ送る。
MVP CivitAI LoRA bridge Animadexの loras[] を既存CivitAI APIへ橋渡しし、詳細表示、外部ページopen、download候補化を行う。 ダウンロードは既存の安全境界とmanifestを使う。
P1 Prompt Composer / Pro Prompt Slots character triggerを主題slot、外観タグを服/小物・表情/ポーズslot、artistを質感/画風slotへ分配する。 Anima向けprompt順は既存のモデル別slot orderへ追加する。
P1 Model Library / Preflight checkpoint prompt profileに Anima familyを追加する場合、Animadex検索を関連キャラ/LoRA探索として表示。 現行Forgeでは「Anima用知識であり生成互換は別」と明示する。
Later ローカルmirror/import フルDBまたはCSV公開後、userdata/animadex/ にローカルインデックスを作る。 サイト負荷とAPI変更リスクを下げ、offline検索を可能にする。

5. 非目標

  • 現行 runtime/forge で公式Anima Baseを直接生成可能にすることは、この機能の非目標。
  • Animadexの全画像を自動ダウンロード、転載、再配布することはしない。
  • キャラクター再現の正確性を保証しない。候補カードに「sample-based / verify visually」を明示する。
  • 起動時にAnimadexやCivitAIへ自動大量アクセスしない。検索はユーザー操作時だけ。
  • 商用利用可否の判断を自動化しない。Anima本体と派生モデルの非商用条件は別警告にする。

6. 既存機能とのシナジー

既存機能接続案価値
Prompt Helper 自然文やキャラ名からAnimadex候補を検索し、positive / negative / warningの横へ「外部知識候補」を追加。 ローカルルールだけでは弱い固有名詞・作品名・衣装タグを補える。
Prompt Composer 検索結果のタグをslot単位で挿入。品質/Score、主題、服/小物、質感/画風へ分配。 ただタグを末尾に貼るのではなく、Yoitomoshiのprompt構造へ吸収できる。
Prompt Library / Tags Workspace よく使うAnimadex候補をcustom libraryへ保存。source.kindは既存型へ animadex 追加を検討。 毎回ネット検索せず、制作語彙として育てられる。
CivitAI search / download AnimadexのLoRA URLからmodel idを取り、CivitAI APIで詳細・primary file・license系metadataを再取得。 LoRA候補探しと安全なdownload manifestを一つの流れにできる。
Model Library Anima系checkpoint / LoRAに「Animadexでキャラ探索」ボタン。関連LoRAをcheckpoint profileへ保存。 モデル管理から制作準備までがつながる。
History / Next Image 生成履歴にAnimadex source metadataを保存し、同じキャラ・作品・artistで再探索できるようにする。 「この方向で続ける」の次の一枚に外部キャラ知識を足せる。
Forge Neo / ComfyUI V2構想 Anima familyのpreflightとして、3ファイルbundle、推奨prompt order、Animadex検索をまとめる。 将来Anima生成経路を作った時に、UIの知識層を再利用できる。

7. 推奨アーキテクチャ

1. Renderer UI Prompt Helper / Composer / Model Libraryから検索要求
2. IPC animadex:search で型付き境界を通す
3. Main client electron/animadex-api.ts がfetch、normalize、cache
4. Suggestion adapter trigger/tags/lorasをYoitomoshi用候補へ変換
5. Existing actions Prompt挿入、slot挿入、CivitAI詳細取得、download

7.1 追加ファイル案

ファイル役割
electron/animadex-api.ts Animadex HTTP client。baseUrl、timeout、cache、schema normalize、error分類、rate-limit backoff。
src/shared/types.ts AnimadexCharacterResult、AnimadexArtistResult、AnimadexLoraRef、AnimadexSearchRequest、AnimadexSearchResult を追加。
src/shared/ipc-channels.ts animadexSearch、animadexFacets、将来用 animadexOpenExternal など。
electron/preload.ts window.api.animadex.search() を公開。
electron/ipc-handlers.ts 入力検証、設定読み込み、client呼び出し、cache保存。外部URLは許可ドメインを限定。
src/lib/animadex-suggest.ts Animadex結果をPrompt Composer slot、Prompt Helper候補、CivitAI bridge候補へ変換する純粋関数。
src/components/AnimadexSearchPanel.tsx Prompt Helper / Model Libraryから再利用できる検索カードUI。data-testidを固定。

7.2 cache設計

userdata/animadex/cache.json またはquery単位の小ファイルに、 query hash、mode、params、fetchedAt、expiresAt、result summaryを保存する。 既定TTLは検索結果24時間、facets 7日、失敗時はstale cacheを警告付きで返す。 画像本体は自動保存せず、thumbnail URLだけ保持する。

8. UI要件

画面表示操作
Prompt Helper 入力欄の下に「Animadex」検索セクション。mode切替は Characters / Artists / Series。 結果カードにthumb、name、series、count、trigger、tags、LoRA有無、source badge。 triggerを追加、tagsを追加、slotへ送る、 CivitAI LoRAを見る、Danbooruを開く。
Prompt Composer slot draftの横にAnimadex候補chipを出す。artistは @artist name を明示。 slot別挿入、全slotプレビュー、Anima prompt orderで整形。
Model Library Anima系またはAnima LoRA候補のカードに「Animadexで探索」。 関連LoRAをcheckpoint profileの relatedModels.loras へ保存。
Preflight 現行ForgeでAnima用候補を使う場合は「知識参照のみ」の警告。Forge Neo / ComfyUIならAnima bundle確認へ昇格。 不足しているText Encoder / VAE / Anima checkpointを別runtimeの準備項目として表示。

9. 受け入れ条件

  1. hatsune miku をCharacters検索し、trigger、series、tags、thumbnailを表示できる。
  2. asuka をLoRAありで検索し、Animadex由来LoRAを表示し、CivitAIページを開ける。
  3. 検索結果の trigger と tags を既存promptへ追記でき、既存promptを上書きしない。
  4. Prompt Composer slot挿入ONの場合、結果が主題・服/小物・質感/画風などのslotへ入る。
  5. Animadexが落ちている、schemaが変わった、rate-limitされた場合でもアプリ全体は壊れず、stale cacheまたは明確なエラーを返す。
  6. CivitAI downloadは既存download manifest、allowed domain、SHA検証の流れを使い、Animadex URLから直接任意ファイルを保存しない。
  7. 起動時にはAnimadexへアクセスしない。ユーザー検索または明示更新時だけ通信する。
  8. DOM QAは表示文言ではなく data-testid で検索、結果カード、挿入、エラー状態を確認する。

段階作業検証

1. IPC client animadex-api.ts、shared types、IPC、preloadを追加。cacheとtimeoutを実装。 npm.cmd run typecheck、qa:dom:api。

3. Prompt adapter animadex-suggest.ts でtrigger/tags/artistをPrompt Composer slotへ分配。 prompt-composer、prompt-helper-review 系DOM QA。
4. CivitAI bridge LoRA URLからmodel idを解析し、既存CivitAI APIで詳細取得、download候補化。 CivitAIページopen、download fixture、source metadata保存確認。

11. リスクと停止条件

リスク対策停止条件
Animadex APIが公式安定契約ではない main process clientに閉じ込める。schema normalizeとcache fallback。baseUrlを設定化。 APIが頻繁に変わり、1週間以内に追従不能な破壊変更が続く場合はlocal dataset待ちへ切替。
サイト負荷・rate limit 起動時アクセス禁止、debounce、1 query cache、manual refresh、User-Agent明示。 運営がAPI利用を明示的に拒否した場合は即停止し、ブラウザopenだけにする。
LoRA fuzzy matchの誤一致 CivitAI側のmodel id、title、baseModel、trainedWordsを再確認し、ユーザー承認後にdownload。 誤一致率が高く、キャラ名だけで安全に提示できない場合は「外部ページを開く」に縮小。
作品・キャラ・絵師名の権利/規約 候補はprompt補助であり、商用可否を保証しない。Anima非商用警告をPreflightへ出す。 公開配布や商用制作サポートへ広げる場合はライセンス/利用規約の別レビューを必須化。
現行Forgeとの誤解 「Animadex候補はAnima向け知識。現行Forgeで再現保証なし」とUIに出す。 ユーザーがAnima生成機能と誤認するUIになる場合はラベルと導線を再設計。

12. 推奨MVP仕様

最初の実装は、Prompt Helper内の折りたたみパネルとして始める。 トップレベルタブを増やさず、既存のPrompt/Model/CivitAI導線を強くする。 成功したらPrompt Composer slot挿入とModel Library profileへ拡張する。
  • 検索UI: mode、query、LoRAありのみ、sort、page。
  • 候補カード: thumb、name、series/artist、count/score、trigger、top tags、LoRA count、source links。
  • 挿入操作: trigger追加、tags追加、slotへ送る、custom libraryへ保存。
  • CivitAI操作: LoRA詳細取得、CivitAIで開く、既存download modalへ送る。
  • 安全表示: source badge、Anima専用注意、sample accuracy注意、非商用注意。
  • 保存: query cache、最近使ったAnimadex候補、history source metadata。
