Yoitomoshi Forge Studio AIエコシステム横断調査

Stable Diffusion / Forge / ComfyUI / 拡張機能 / 最新モデル / LoRA・LyCORIS / VAE / Civitai / AIイラストコミュニティから、Yoitomoshi Forge Studioに取り込める要素を 実装候補へ変換した調査レポート。確認日は 2026-05-22。

結論

いま最も費用対効果が高いのは、Forgeの生成本体を大きく作り直すことではなく、 モデルファミリー別の制作プロファイル、LoRA/タグ/翻訳の知識化、 Civitaiメタデータの検証付き取り込み、ComfyUI側への新世代モデル避難経路を整えること。

VAE / Decoder / 速度と画質

項目 確認情報 Yoitomoshiでの扱い
SDXL VAE fp16 fix madebyollinのSDXL-VAE-FP16-Fixは、SDXL VAEをfp16でNaNを出さず動かす目的の派生VAE。 sdxl-vae-fp16-fix SDXLプロファイルで推奨候補に出す。--no-half-vaeが必要なケースを検知して説明する。
TAESD / AutoencoderTiny Diffusers DocsはTAESDをStable Diffusion VAEの小型蒸留版として、ほぼ即時のlatent decodeに使えると説明。 AutoencoderTiny 本番出力ではなく、プレビュー高速化や履歴一覧の軽量サムネイル検証候補にする。
FLUX / SD3.5専用構成 FLUX.2やSD3.5はモデルごとにテキストエンコーダ、VAE/decoder、量子化ファイル、フィルタ条件が異なる。 「Checkpointだけ入れれば動く」というUIにしない。必要ファイルをpreflightで一覧化する。

機能 取り込むデータ UI 安全策
モデルカード取り込み モデル名、Version名、Base Model、creator、description、trigger words、recommended resources、files/hash、license。 Model Libraryの詳細パネルに「Civitai由来」セクション。 hash一致時だけ「このローカルファイルに対応」と表示。
推奨プロンプト抽出 description内の明示的なPositive/Recommended/Example promptセクション。 Prompt Helperに「モデル推奨」チップとして表示。 自由文から勝手に発火語を作らない。既存のdry-run previewを維持。
画像メタデータ学習 公開画像のprompt、negative、sampler、steps、CFG、seed、resources。 「参考画像から設定を読む」ビュー。保存前に出典とNSFW/ライセンス注意を表示。 API/HTML構造変更に備えてparserを分離。ユーザーのCivitai APIキーは環境変数または安全な設定ストアへ。
タグトレンド タグ名、カテゴリ、件数、モデルファミリー、期間。 タグライブラリの「候補」タブ。 人気語だけに寄せすぎない。代替タグと探索テンプレートを併記。

AIイラストコミュニティからの学び

観察 根拠 実装へ変換
タグ体系はモデルと文化圏で違う Danbooru系はcharacter/copyright/artist/general/meta/ratingのような分類を持ち、Pony/Illustrious/NoobAIはこの文化圏の影響が大きい。 Promptタグ管理に「人物/衣服/構図」だけでなく、model-family、source、rating、quality、artist-styleを持たせる。
人気プロンプトは均質化を招く Civitai由来の600万超プロンプト分析は、人気タグの反復でプロンプトが均質化し、視覚的多様性とも相関すると報告。 arXiv:2504.14125 「定番」だけでなく「探索」「差分」「反対方向」チップを出す。履歴レビューで似た出力が続いたら別系統タグを提案。
LoRA運用では発火語の分類が重要 Civitai/Redditの実務投稿では、base trigger、helper words、negative words、version別管理の需要が強い。 LoRA詳細に「必須」「補助」「避ける」「weight候補」を分けて保存。生成前に不足をチェック。
二次情報は速いが壊れやすい Reddit/ブログは実機トラブルや運用ノウハウの発見に強いが、仕様断定には向かない。 レポート内では「一次情報」「コミュニティ実務」「未検証」を分ける。実装時は公式DocsまたはローカルQAで確定する。

実装順序

  1. 短期: 既に始まっているタグチップ辞書追加・翻訳・お気に入り化を安定化し、設定画面に翻訳プロバイダ選択を露出する。
  2. 短期: Model Libraryへモデルファミリーの手動分類と推奨prefix/negative/解像度を持たせる。
  3. 中期: Civitai/LoRA metadataからtrigger wordsとrecommended promptsを取得し、dry-run previewで保存する。
  4. 中期: VAE/decoder preflightを追加し、SDXL fp16 fixやTAESD系を用途別に案内する。
  5. 中期: ControlNet/ADetailerを「タスクテンプレート」へ変換し、初心者が拡張名を知らなくても使えるようにする。
  6. 後期: FLUX.2/SD3.5はComfyUI V2に安全なワークフローパックとして追加し、Forge側にはモデル導入・実行経路だけを出す。

未確定・採用保留

  • 「Stable Diffusion 4」系の記事は、今回の確認範囲ではStability AI一次情報で裏取りできなかったため採用しない。
  • Civitai APIは公式ドキュメント移行中。endpointの長期安定性を前提にせず、取得失敗時のfallbackとキャッシュを実装する。
  • FLUX.2は魅力が大きいが、Forge直結は未検証。最初はComfyUI V2でワークフロー検証する。
  • 無料翻訳は規約・レート制限・品質差がある。自動翻訳は下書き扱いにし、ユーザー編集を正とする。
