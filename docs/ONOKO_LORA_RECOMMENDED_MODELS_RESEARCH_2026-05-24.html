ONOKO LoRAにおすすめのモデル調査レポート

2026-05-24時点で、ローカルONOKO LoRAの埋め込みメタデータ、Civitai API、Hugging Faceモデルカード、PromptHero、 Tensor.Art、SeaArt、PixAI記事を横断して、実用優先のcheckpoint候補を評価した。

調査日: 2026-05-24 対象: ONOKO / 0n0k0 LoRA 現物判定: SDXL / Illustrious系 成果物: docs HTML
結論

最優先は n4mik4ILSFWNSFW_v20。次点で WAI-illustrious-SDXL v17

ローカルの現行ONOKO LoRAは、safetensorsヘッダー上で n4mik4ILSFWNSFW_v20.safetensors を学習元checkpointとして記録している。 そのため、忠実度確認・採用カット作成・LoRA weight調整の基準モデルは n4mik4__- IL -__SFW/NSFW_ v2.0 で固定するのが最も合理的。

量産時の安定性や、より一般的なIllustrious系の質感を見たい場合は WAI-illustrious-SDXL v17.0 を2本目の比較モデルにする。 Animagine XL 4.0、Pony Diffusion V6 XL、NoobAI系は有力だが、ONOKO LoRAの学習元からは外れるため、 「本命」ではなく比較・再学習判断用に回す。

重要

同じSDXLでも系統差は大きい

SDXL / Illustrious / NoobAI / Pony / Animagine は同じ広義SDXL圏でも、タグ作法と内部分布が違う。 キャラLoRAは顔・衣装・色が崩れやすいので、まず学習元と同じcheckpointで基準画像を作る。

推奨順位

1n4mik4__- IL -__SFW/NSFW_ v2.0

用途: ONOKO LoRAの顔・衣装・色・アクセサリー再現の基準。 ローカルLoRAの学習元そのものなので、まずこれで基準seedを作る。

根拠: Civitai model ID 2366670、base modelはIllustrious。 PromptHeroにも同一ファイル n4mik4ILSFWNSFW_v20.safetensors と sdxl_vae.safetensors が掲載され、LoRA stackingは低weightが良い、通常CFG 4.5という説明がある。

ONOKO忠実度: 最上位 汎用性: 中 導入: 済み

2WAI-illustrious-SDXL v17.0

用途: 量産・綺麗なIllustrious系・現在の人気checkpointでの実用比較。 Civitai API上ではdownloads 1,364,569、thumbs up 81,375と非常に大きい実績がある。

注意: Civitai上ではNSFW扱い。ONOKOの顔を強く保ちたい場合はLoRA weightを 0.55-0.70から始め、WAI側の画風が勝ちすぎるなら下げる。

ONOKO忠実度: 高 汎用性: 高 導入: 要download

3Illustrious XL / Obsession / NoobAI系

Illustrious公式v0.1はLoRA・Adapter学習の土台として明記されているが、美観調整済みの量産モデルではない。 ObsessionやNoobAIは強いが、v-pred / eps-predや推奨設定差を確認してから使う。

4Animagine XL 4.0 Opt

公式HFモデルカードが充実し、アニメキャラの清潔感と動きに強い。 ただしONOKO LoRAの学習元ではないため、互換テストではLoRA weightを低めから始める。

5Pony Diffusion V6 XL

すばらしいcheckpointだが、Pony系はprompt templateも分布も別物。 ONOKOをPonyで本格運用したいなら、既存LoRA流用よりPony向けLoRA再学習を推奨。

サイト別の見立て

Civitai

最も実用的。base model、version、download数、thumbs up、ファイル名、hash、license系の情報を取得できる。 今回の推奨順位はCivitai APIの現行メタデータを主軸にした。

Hugging Face

公式モデルカード、Diffusers利用、ライセンス、推奨設定の確認に向く。 Animagine XL 4.0、Illustrious XL、SDXL base、SD1.5、Counterfeitの根拠確認に使った。

PromptHero

n4mik4 v2.0のファイル名、更新日、説明文を確認できた。 Civitai IDに対応するミラーとして、ファイル同定の補助に使える。

Tensor.Art

オンライン生成での人気やリプリント状況を見る用途に向く。 n4mik4、Pony、WAIなどの掲載を確認したが、配布元としてはCivitaiやHFを優先する。

SeaArt

WAI系のオンライン生成実績・モデル設定例を見るのに有効。 ただし表示上のbase model分類がCivitaiと揺れることがあるため、ローカル導入判断では補助扱い。

PixAI

SDXL anime modelの使い分け記事が参考になる。 Animagineは動的キャラ、Illustriousは幅広いアニメスタイル・既存キャラ認識という整理は、ONOKO比較にも使いやすい。

推奨テスト手順

1. n4mik4で基準seedを作る

Checkpoint:
n4mik4ILSFWNSFW_v20.safetensors

Positive:
<lora:onoko_0n0k0_sdxl_v3_2_patch_b:0.65>,
0n0k0, solo, looking at viewer, eyewear on head, sunglasses,
white jacket, skirt, smile, clean lineart, detailed eyes,
masterpiece, best quality

Negative:
worst quality, low quality, bad anatomy, bad hands,
extra digits, text, watermark, logo

Settings:
832x1216, Euler a, 28 steps, CFG 4.5

2. checkpointだけ差し替えてA/B

  1. seed、prompt、negative、LoRA weightを固定する。
  2. n4mik4、WAI v17、Animagine XL 4.0、必要なら NoobAI/Obsession を同条件で比較する。
  3. 顔の同一性、眼鏡・ジャケット・配色、手、背景、線の濁りを5段階で採点する。
  4. ONOKOの特徴が崩れるcheckpointは、画質が良くても「画風用」として二軍にする。

採用・不採用の判断

今すぐ使う

  • n4mik4ILSFWNSFW_v20 を基準モデルに固定。
  • onoko_0n0k0_sdxl_v3_2_patch_b をLoRA本命にする。
  • 初期weightは0.65。崩れるなら0.55、弱いなら0.75まで。

追加download候補

  • WAI-illustrious-SDXL v17.0: 量産用比較。
  • Animagine XL 4.0 Opt: 清潔なアニメ画風比較。
  • Obsession / NoobAI: Danbooruタグ・照明の比較。

後回し

  • Pony Diffusion V6 XL: Pony版ONOKOを作る時に再評価。
  • SD1.5 checkpoints: SD1.5 ONOKO LoRAがある時だけ。
  • Anima / Flux / Z-Image系: 現行ONOKO LoRAとは別アーキテクチャ。

最終メモ

ONOKO LoRAの現行ファイルを見る限り、モデル探しの中心は「人気checkpoint」ではなく 「Illustrious系、かつn4mik4に近いcheckpoint」である。最初の採用フローは、 n4mik4 でキャラの正解を作る、WAI で量産品質を確認する、 Animagine / NoobAI / Pony は別系統として崩れ方を記録する、の順が最短。
