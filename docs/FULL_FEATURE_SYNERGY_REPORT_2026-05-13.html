結論

現在の最重要テーマは、新機能をさらに単体追加することではなく、すでにある History、Workspace、Prompt Helper、ControlNet Builder、 FABRIC、Regional Prompter、Model Library を「制作レシピ」として束ねること。

既存実装は広いが、入口が分かれているため、初心者には「次にどれを押せばよいか」がまだ分散して見える。 次はレシピ復元、プリフライト、視覚ビルダー、履歴活用を一本化すると伸びる。

今回の判断

既存基盤は強い Storage/IPCは整備済み 導線統合が課題 Recipe保存に差分あり 実機QAは別途必要

追加すべき中心機能は「Generate前の制作意図を保つ」方向。ControlNet、Regional、FABRIC、LoRA、Promptタグを別々に強化するより、同じ素材制作フロー内で往復できるようにする。

1. 総合評価

アプリは Forge を API専用で裏起動し、Electron/React UI から生成、素材管理、Civitai/Hugging Face検索、拡張機能、Upscale、ワークスペース保存を操作する設計。 Stable Diffusion WebUI の汎用UIを置き換えつつ、ゲーム素材制作で必要な「生成結果を次工程へ送る」導線が多く実装済み。

01
Prompt
Prompt Library、Quick Preset、英訳、Prompt Helper、タグ編集で作る。
02
Control
ControlNet、ADetailer、Regional Prompter、FABRICで構図や品質を固定する。
03
Generate
txt2img/img2img、inpaint、variation、progress previewで候補を作る。
04
Reuse
History、metadata解析、send to img2img/Upscale、Workspaceで再利用する。
05
Manage
Model Library、Download jobs、Inspector、Merger、Converterで環境を保守する。

3. 既存機能の全棚卸し

3.1 起動・設定・Forge制御

  • Forgeパス、ポート、自動起動、追加引数、Civitai APIキー、UI言語を設定モーダルで保存。
  • Forge start/stop/status、ready待ち、progress preview、interruptをIPC経由で操作。
  • API専用起動、不要Gradio系拡張の自動無効化、stale process cleanup、single instance lock。
  • 起動診断として app ready、IPC ready、renderer loaded、Forge ready を startup-metrics.jsonl に記録。

3.2 生成ワークフロー

  • txt2img と img2img の両対応。
  • Sampler、Scheduler、Steps、CFG、Size、Seed、Batch、Iterations、Clip Skip、VAE、Denoiseを管理。
  • LoRAはPrompt内トークンを整理し、Active LoRAから最終プロンプトへ合成。
  • 生成結果はHistoryへ保存し、実seed、model、VAE、LoRA、sampler、size、denoiseを記録。
  • 拡張ガードで、FABRICとControlNet referenceの競合、Regional prompt/ratio不一致を生成前に止める。

3.3 Prompt入力・タグ操作

  • PromptEditor、オートコンプリート、トークンカウンター、シンタックスハイライト、重み調整ショートカット。
  • PromptTagChipsでタグを分解し、重み変更、削除、重複整理、並べ替え、Positive/Negative移動、選択タグ保存が可能。
  • QuickPresetBarでPositive/Negative別の短いスニペットをトグル、保存、非表示、削除。
  • 自然文から英語タグへ変換する translatePromptToEnglishTags と Prompt Helperを搭載。

3.4 Prompt Library

  • resources/prompt-library.ja.yaml を組込ライブラリとして読み込み。
  • カテゴリ、サブカテゴリ、タグを表示。クリックで追加、Shiftで強調、AltでNegativeへ追加。
  • お気に入り、Recent、検索、サブカテゴリ複数選択、巨大カテゴリの段階表示に対応。
  • ユーザー追加カテゴリ/グループ/タグを custom-prompt-library.json に保存。組込カテゴリへの追加はshadow merge。

3.5 img2img・入力画像・Inpaint

  • 画像D&D、クリック選択、Ctrl+V貼付、Previewへのドロップで入力画像へ設定。
  • DeepDanbooru/CLIP interrogateでタグ抽出し、クリックでPromptへ追加。
  • Inpaint maskキャンバスで白領域を塗り、Forge img2imgのmaskとして送信。
  • PNG/JPEG/WebPメタデータ抽出、A1111形式テキスト貼付解析、Civitai照合に対応。

3.6 Preview・Metadata・次工程アクション

  • 生成中current_imageを表示し、sampling stepと進捗バーを表示。
  • 結果をDownload、Folder表示、img2imgへ戻す、Upscaleへ送る、Variationを開く、Recipe保存。
  • MetadataInfoPanelで現在画像のmodel/VAE/LoRA/params概要を表示。
  • ドロップ画像はCivitai照合結果から不足モデル/LoRA/VAEのDL導線を出す。

3.7 History

  • 最大500件の生成履歴を userdata/history に保存。
  • 検索、期間、model、sampler、LoRA、labelで絞り込み。
  • ラベルは favorite、candidate、rejected、asset。
  • 復元でPrompt、Negative、params、model、VAE、Active LoRA、実seedを戻す。
  • 2件比較でPrompt差分を表示し、履歴画像をUpscaleへ送れる。

3.8 Variation

  • Seed、CFG、Denoiseを軸に2/4/8枚の候補生成。
  • 候補はHistoryへ保存し、サムネイルからPreview表示、img2img/Upscaleへ送れる。
  • img2img時のみDenoise variationを許可し、入力画像必須のガードあり。

3.9 LoRA

  • ForgeのLoRAフォルダを走査し、検索、互換、Favorites、Activeでフィルタ。
  • Civitai hash lookupでthumbnail、trigger words、baseModel、tagsを取得。
  • Active LoRAはweightとtrigger wordsを管理し、生成時にプロンプトへ合成。
  • 自動提案はCivitai推奨、base model一致、trigger一致、favorite、直近使用、類似prompt、同カテゴリpenaltyを加味。
  • LoRA使用履歴を lora-usage.json に記録。

3.10 Civitai / Hugging Face

  • Civitaiモデル検索、sort、type、NSFW、人気タグ、cursor/page pagination。
  • Hugging Face検索とファイル選択ダウンロード。
  • Checkpoint/LoRA/VAE/ControlNet等の配置先を自動決定。
  • download progress、cancel、resume、manifest、SHA照合、metadata/preview cache。
  • Checkpointの推奨設定、trained words、recommended VAE/LoRA、community image mining、model update check。

3.11 拡張パネル

  • Dynamic Thresholding: mimic scale、threshold percentile、詳細mode群。
  • FreeU: b1/b2/s1/s2、start/end step。
  • ADetailer: face/hand/person/YOLO-world、unit最大4、prompt/negative/detection/inpaint params。
  • ControlNet: 最大3 Unit、image、module、model、weight、guidance、pixel perfect、resize/control mode。
  • ControlNet Builder: 画像ドロップ、role preset、preprocessor実行、preview、Unit反映、run and apply。
  • Regional Prompter: Columns/Rows、ratio/base ratio、common/base/region prompts、BREAK prompt生成、validation。
  • FABRIC: positive/negative feedback画像、last/input/upload追加、strength、negative weight、start/end、TOME、burnout protection。

3.12 Upscale

  • Simple: Forge extras extra-single-image で非diffusion拡大。
  • Diffusion: MultiDiffusion / Mixture of Diffusers、tile size/overlap、denoise、prompt共有。
  • Ultimate: Ultimate SD upscale script、seams fix、mask blur、padding、redraw settings。
  • Tile ControlNet: upscale専用のControlNet payloadを生成し、通常PromptPanelのControlNetと分離。
  • メタデータ/寸法から推奨設定を提案し、Denoise 0.25/0.35/0.45 と Tile ON/OFF比較を保存可能。

3.13 Tools

  • Workspace: embed、references、settings-only で .yoitoart 保存/復元。
  • Startup Diagnostics: 起動計測サンプル表示。
  • Catalog Health: Forge/model/LoRA/VAE/ControlNet/Upscaler状況とTile model/module確認。
  • Model Library: scan/rescan、検索、種別集計、SHA保持、preview/metadata、download jobs。
  • Model Health: models配下の空ファイル、重複、種別違い、欠落フォルダなどを診断。
  • Model Inspector: safetensors/ckptのヘッダ、tensor key、metadataを表示。
  • Model Merger: Forge Python経由、事前見積り、空き容量、上書き防止、progress log、cancel、自動inspect。
  • Format Converter: ckpt/pt/pthをsafetensorsへ一時ファイル経由で安全変換し、出力inspect。

3.14 多言語・ショートカット・外部リンク

  • src/lib/i18n.ts にja/en/ru/pt辞書。fallback chainは requested -> en -> ja -> raw key。
  • ShortcutsModalで生成、PromptEditor、Library操作、画像貼付/ドロップを案内。
  • タイトルバーからCivitai、Civitai Green、Hugging Face、aipictors、Lexicaへ外部ブラウザで遷移。
  • 外部URLはallowlistで制限。

4. 機能間シナジー

組み合わせ現在できること強化すると何が起きるか
Prompt Helper + Prompt Library + 英訳 自然文から英語タグ候補を作り、組込/ユーザータグも拾える。 自然文を「Promptレシピ」に変換し、Regional領域、LoRA候補、ControlNet役割まで同時提案できる。
History label + FABRIC 履歴に採用候補/没/素材化済みを付けられる。FABRICはpositive/negative画像を受け取れる。 ラベル付き履歴からpositive/negative feedbackを自動構築すれば、好みの方向性を即座に再生成へ反映できる。
History + Workspace Historyは生成条件、Workspaceは拡張設定や画像参照を保存できる。 履歴復元をWorkspace復元相当に拡張すると、Regional/FABRIC/ControlNet込みの完全Recipe再現になる。
Metadata解析 + Civitai DL + Model Library 画像からmodel/LoRA/VAEを推定し、不足資産をCivitaiからDLできる。 画像ドロップだけで不足モデルの検出、ダウンロード、Library index登録、Workspace保存まで一気通貫になる。
Civitai Community Stats + QuickPreset 頻出prompt phrase、sampler、steps、CFG、VAE、LoRAをRecommendationCardで見られる。 モデル別QuickPresetやNegative presetを自動生成すれば、モデルごとの初期設定が大幅に速くなる。
ControlNet Builder + Upscale Tile ControlNet Builderはpose/depth/lineart等の入力生成、UpscaleはTile ControlNetでdrift抑制。 「通常生成用ControlNet」と「Upscale保持用Tile」を同じ素材から連続設定できるようになる。
Regional Prompter + PromptTagChips Regionalは領域Promptを持ち、PromptTagChipsはタグ単位編集できる。 選択タグを領域へ振り分け、BREAK/ADDROW/ADDCOLを視覚的に組む地域Promptビルダーへ進化できる。
ADetailer + Inpaint + History ADetailerは自動顔/手補正、Inpaintは手動mask、Historyは候補比較を保持。 顔だけ/手だけ/衣装だけの修正結果を履歴で比較し、採用候補をFABRIC/Workspaceへ戻せる。
Download Manager + Update Check + Health Scan 未完了DL復旧、SHA、metadata、preview、更新検出、健全性チェックがある。 モデルカードに「未検証/更新あり/壊れた/復旧可能」を統合表示すれば、Toolsを探さず保守できる。
Startup Metrics + Lazy Loading 起動計測と各Tab lazy chunk化は実装済み。 Forge起動ログ、拡張ロード時間、Civitai遅延処理を同じ診断表へ入れると、重い原因を継続的に潰せる。

5. 弱点と取りこぼし

Recipe保存導線が2種類あり、保存範囲が違う

ToolsのWorkspace保存は regionalPrompter と fabric、画像参照モードまで含む。 一方、Previewの「Recipe保存」は簡易snapshotで、Regional/FABRICや参照保存モードが揃っていない。 ユーザー視点では同じ「Recipe」なので、ここは統一すべき。

History復元は強いが、拡張状態までは戻らない

Historyはprompt/model/VAE/LoRA/paramsを戻せるが、ControlNet、Regional、FABRIC、ADetailerなどの拡張状態はWorkspace側の責務。 「良かった生成を完全再現する」体験にはHistoryとWorkspaceの統合が必要。

Regional Prompterはまだ1D寄り

Columns/Rows、ratio、BREAK prompt生成はあるが、2D gridを直接編集するADDROW/ADDCOLビルダーではない。 画面内の領域を見ながらPromptを割り当てるUIが入ると、初心者にも使いやすくなる。

FABRICは履歴・ラベルとの連携が未統合

現在はlast/input/uploadからfeedback画像を入れる形。Historyの「採用候補」「没」ラベルを直接FABRICへ流せると、ユーザーの評価が生成制御へ戻る。

ControlNet Builderは単発Unit中心

Role presetとpreprocessor実行は強いが、pose+depth+reference、lineart+tileのような複合レシピはまだ手作業。 よく使う組み合わせを「構図固定」「画風参照」「高解像保持」として保存すると効果が高い。

Model Libraryの情報がToolsに寄っている

index、download job、integrity、update、previewは存在するが、TitleBar/LoRA/Civitai検索/Recommendation側で一枚のモデルカードとして見えるわけではない。 生成中の判断場所へ状態を出す必要がある。

注: 今回は静的精査のため、実Forge生成、Civitai live API、ブラウザ目視QAは実施していない。実動作の最終判断は既存QAに続く再確認が必要。

6. 改善案と優先順位

P0-1

Recipe保存の統一

Previewの「Recipe保存」をTools Workspace保存と同じsnapshot builderへ寄せ、Regional/FABRIC/画像参照モードも保存する。

効果: 生成直後に保存したRecipeが、Toolsで保存したWorkspaceと同じ信頼性になる。
P0-2

Historyから拡張込みRecipeを作る

History itemにWorkspace snapshot IDを紐づける、または生成時の拡張snapshotをHistory paramsへ軽量保存する。

効果: 「良かった1枚」からControlNet/Regional/FABRIC込みで再開できる。
P0-3

Generation Preflight Panel

生成ボタン直前に、model/VAE/LoRA/ControlNet model/Regional ratio/FABRIC競合/Upscale tile modelを一覧で判定する。

効果: 失敗理由を生成後のエラーではなく、押す前に見える化できる。
P0-4

History label to FABRIC

Historyの favorite/candidate をFABRIC positive、rejected をnegativeへ一括追加する。

効果: ユーザー評価を直接生成制御へ戻せる。今ある機能同士の接続だけで価値が高い。
P0-5

Model card統合

TitleBarのモデル選択、Recommendation、Model Library、Update Check、Health Scanを統合したモデル状態カードを作る。

効果: 「このモデルは更新あり/破損なし/推奨VAEあり/よく使うLoRAあり」が一目で分かる。
P1-1

Regional 2D Grid Builder

行/列追加、比率調整、領域選択、Prompt割当、BREAK/ADDROW/ADDCOL出力を視覚UI化する。

効果: Regional Prompterが専門ユーザー向けから日常操作へ近づく。
P1-2

ControlNet recipe presets

pose+depth、lineart+reference、tile upscaleなど複数Unitの組み合わせを保存/適用できるようにする。

効果: ControlNet Builderの単発入力生成が、制作用途別の構図レシピになる。
P1-3

Civitai stats to QuickPreset

community miningの頻出positive/negative phraseをモデル別QuickPresetとして保存する。

効果: モデルごとの当たりPromptを手元のプリセットへ蓄積できる。
P1-4

Inpaint workspace upgrade

消しゴム、mask反転、blur preview、領域履歴、History候補比較を追加する。

効果: ADetailerで直らない部分を手動修正しやすくなる。
P1-5

Upscale比較レポートの自動HTML化

保存済み比較候補と判断基準から、denoise/tile ON/OFFの差分HTMLを自動生成する。

効果: 素材ごとのdrift/seam判断を後から見返せる。
P2-1

History/Library indexing強化

500件を超えたらSQLiteまたは軽量indexへ移行し、タグ検索、類似Prompt、モデル別時系列を高速化する。

効果: 長期運用で履歴が資産化する。
P2-2

拡張互換性モニタ

Forge拡張のscript title、args順、API endpoint availabilityを起動時に軽く検査する。

効果: 拡張更新でpayloadが壊れた時、生成失敗前に検知できる。
P2-3

Prompt翻訳辞書のユーザー編集

自然文から英語タグへの変換辞書に、ユーザーの言い回しと固定タグを追加できるようにする。

効果: 外部AIなしのPrompt Helperを個人制作向けに育てられる。

7. ゴールまでの実行順

  1. Recipe保存統一: Workspace snapshot builderを共通化し、Preview保存でもRegional/FABRIC/参照保存を扱う。
  2. History to FABRIC: 履歴ラベルからpositive/negative feedbackを一括追加するUIを作る。
  3. Generation Preflight: 生成前の不足/競合/不整合を1パネルで表示する。
  4. Regional 2D Grid Builder: 既存Regional Panelを2D領域編集へ拡張する。
  5. ControlNet recipe presets: Builderのrole presetを複合Unitレシピへ広げる。
  6. Model card統合: TitleBarとToolsのModel Library情報をつなぐ。
  7. Upscale比較HTML: 保存済み比較からdrift/seam判断レポートを生成する。

次に着手するなら

最初の実装対象は Recipe保存統一 が最もよい。 理由は、既存のTools Workspace保存がすでに正しい保存範囲を持っており、Preview側の簡易保存との差を埋めるだけで、 History、FABRIC、Regional、ControlNetの後続改善すべての土台になるため。
