Extension synergy implementation

Regional Prompter / FABRIC をアプリ作業面へ接続

GitHub で確認した拡張仕様をもとに、Prompt パネルから領域別プロンプトと画像フィードバックを扱えるようにした。 どちらも Forge / A1111 の alwayson_scripts に流すため、既存の txt2img / img2img 生成フローと同時に使える。

調査元: hako-mikan/sd-webui-regional-prompter, dvruette/sd-webui-fabric

Before

課題

  • Regional Prompter は WebUI 側で手入力する前提で、アプリの prompt 作成と分断されていた。
  • FABRIC は liked / disliked 画像を拡張フォルダへ置く必要があり、履歴画像や入力画像から直接渡せなかった。
  • Workspace 保存に、領域プロンプト設定や FABRIC 画像参照をまとめて残す導線がなかった。
After

今回の到達点

  • Prompt パネルに Regional Prompter と FABRIC feedback を追加した。
  • 領域プロンプトは BREAK 区切りのメインプロンプトへ反映し、同時に AlwaysVisible script を有効化する。
  • FABRIC は直近生成、入力画像、手元画像を positive / negative 側へ登録し、拡張の log/fabric/images へ保存する。
  • Workspace 保存時に Regional Prompter / FABRIC 状態も残し、参照保存では FABRIC 画像ファイルからサムネイル復元できるようにした。

GitHub 調査から実装へ落とした仕様

拡張 確認した仕様 アプリ側の活用
Regional Prompter title() は Regional Prompter。 UI は有効化、Matrix/Mask/Prompt 系の分割モード、ratio、base/common、calc mode、flip などを positional args として受ける。 src/lib/extension-payload.ts で alwayson_scripts["Regional Prompter"] を生成。 UI では左右/上下、比率、common/base、領域リストを最小セットとして先に扱えるようにした。
FABRIC title() は FABRIC。 liked / disliked は画像データ本体ではなく、拡張配下の log/fabric/images にある filename 配列として渡す。 electron/ipc-handlers.ts に保存 IPC を追加し、画像を SHA ベースの filename で FABRIC 拡張フォルダへ保存。 payload は filename 配列と weight / start / end / ToMe / burnout protection を渡す。

既存機能とのシナジー

Prompt Helper

タグ候補から領域別プロンプトへ

既存の append 型 Prompt Helper で作った候補を、Regional Prompter の R1/R2 に振り分ける入口ができた。

History

採用/没の画像を FABRIC へ再利用

直近生成を positive または negative に入れられるので、履歴ラベル運用と FABRIC の liked/disliked がつながる。

Workspace

作業セットとして保存

Prompt、ControlNet、ADetailer、Regional Prompter、FABRIC を同じ Workspace Snapshot に入れられる。
