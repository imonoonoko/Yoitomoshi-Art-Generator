素材の入手方針

  • 最優先は自分で撮影した写真と、自分で生成/制作したキャラPNG。人物の顔が写る場合は本人同意がある素材に限定する。
  • 公開レポートに載せる可能性がある素材は、人物・ブランド・看板・作品が写り込まないものを使う。
  • 外部素材を使う場合、Unsplash License と Pexels License は無料利用を許すが、未加工販売や人物/ブランドの推奨表示には制限がある。QA用途でも出典URLと取得日を manifest.json に残す。
  • Wikimedia Commonsはファイルごとにライセンスが違うため、CC BY / CC BY-SA / Public Domain の条件を個別に確認する。

導入順

  1. P0 ControlNet QAセット: OpenPose、Depth、MistoLine rank256を入れる。容量は約5.8GB。
  2. P0 実素材QA: 夜写真2枚、昼/室内2枚、透明PNGキャラ2枚を自前で用意し、AIキャラ追加のBefore/Afterパッケージを1セット保存する。
  3. P1 輪郭比較: xinsir Canny V2を追加し、MistoLineとの境界/背景保持差を比較する。
  4. P1 参照強化: Forge側にIP-Adapter moduleが出ることを確認してから、IP-Adapter SDXL safetensorsを入れる。
  5. P1 透明生成: LayerDiffuse拡張を導入し、透明PNG生成だけを単独QAする。
  6. P2 照明: IC-LightはForge直接統合ではなく、外部runnerの効果確認から始める。
  7. P2 実験: xinsir ControlNet Union / ProMaxは通常モデルQA後に、API payload差分を調べてから試す。
