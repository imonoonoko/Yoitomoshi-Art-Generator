UI research report / generation workspace

生成画面モード切替UI 調査・改善提案

対象は Yoitomoshi Art Generator の生成画面左カラムにある 作る / 整える / 高度 の切替導線。 タブの押しやすさだけではなく、プロ向けUI設計資料から「そもそもタブで扱うべき情報か」を検証し、 実装へ落とせる改善案に整理した。

作成日: 2026-05-16 対象: src/components/PromptPanel.tsx 目的: タブ切替の扱いづらさ解消 方針: payload仕様は変えない

結論

推奨案: タブ式から「セクションナビ + 折りたたみパネル」へ変更する

作る / 整える / 高度 は排他的なページではなく、1つの生成payloadに同時に効く設定群である。 調査した複数のデザインシステムは、タブを「同じ文脈内で1つのパネルだけを見る」用途に限定し、 比較や同時確認が必要な情報、送信対象が分かりにくいフォームには注意を促している。 そのため、現状の「タブで隠す」構造をさらに磨くより、各モードを同一スクロール面のセクションとして扱い、 上部の小さなナビで該当セクションへジャンプ・展開する形が最も筋が良い。

今すぐ直すべき違和感

モード切替が「表示カテゴリ」なのか「生成に効く設定のON/OFF」なのか直感的に分かりにくい。

根本課題

最小の勝ち筋

タブではなく、全セクションの見出しを常に存在させる。クリック時は表示切替ではなくスクロールと展開にする。

推奨

残すべき現設計

プロンプト欄、タグ並び替え、Generate、Preflight、ActiveFeatureSummary は常時表示のまま維持する。

改善方針

1. タブを「排他的ビュー」から「セクション移動」に変える

GenerationModeTabs は GenerationSectionNavigator に置き換える。 クリック時は generationMode だけを切り替えてパネルを差し替えるのではなく、 対象セクションを展開してスクロールする。これにより全カテゴリの存在が常に見える。

2. セクションヘッダーに状態を出す

高度 が閉じていても ControlNet 1、FreeU ON、ADetailer OFF のように主要状態を出す。 「隠れているが効いている」問題を要約で潰す。

3. 頻繁な操作はプロンプト欄周辺へ固定する

タグ並び替え、Format、翻訳、Generate、Preflight要約は常時表示のまま維持する。 作る/整える/高度 はプロンプト作業を遮らない補助導線に下げる。

4. 上級機能は「ONのもの優先」

高度セクションは全パネルを常に展開しない。まず有効中/警告中のパネルを上に出し、 未使用の拡張は「追加する」リストへ寄せる。プロ向けツールほど状態の見通しが重要。

実装計画

  1. Phase 1: タブをナビ化

    GenerationModeTabs を GenerationSectionNavigator に置換。role=tablist は使わず、通常の button と aria-controls で対象セクションへ移動する。

  2. Phase 2: 3セクションを常時DOMに存在させる

    CreateModePanel、RefineModePanel、AdvancedModePanel を CollapsiblePanel 化し、表示切替ではなく展開状態で制御する。payload/state処理は変更しない。

  3. Phase 3: 要約と警告をセクションヘッダーへ統合

    LoRA数、Dynamic Prompt、ControlNet、ADetailer、FABRIC、Preflight警告数をヘッダーに表示。閉じていても効いている設定が分かるようにする。

  4. Phase 4: PreflightとActive chipsの誘導を更新

    チップや「開く」ボタンは、対象セクションを展開してから対象要素へスクロールする。現在の generationMode 依存を sectionId に寄せる。

  5. Phase 5: DOM QA更新

    qa:dom:generation-modes を qa:dom:generation-sections 相当に更新。全セクションの見出し存在、展開/折りたたみ、チップ誘導、Preflight誘導、Alt+1/2/3を検証する。

受け入れ基準

UX

  • ユーザーが現在の設定カテゴリを見失わない。
  • 閉じたセクションに有効設定がある場合、ヘッダーだけで分かる。
  • Prompt編集、タグ並び替え、Generateは常にすぐ触れる。
  • モード移動はクリック、キーボード、チップ、Preflightから同じ挙動になる。

技術

  • 生成payload、History metadata、Dynamic Prompt解決ロジックは変更しない。
  • 既存の CollapsiblePanel を利用し、独自の複雑な状態管理を増やしすぎない。
  • DOM QAは文言ではなく data-testid と状態属性で検証する。
  • npm.cmd run typecheck、npm.cmd run build、対象DOM QAを通す。

実装時の注意

タブをやめる場合、ARIAの意味も変える

内容を隠すタブパネルではなく、ページ内セクション移動にするなら role="tablist" は使わない。 button aria-controls="generation-section-advanced" のような通常ナビにして、 展開後の見出しへフォーカスまたはスクロールする方が支援技術にも実態に合う。

見た目だけの縮小は逆効果

Figma UI3の事例でも、すっきり見せるために操作をアイコンや浮遊パネルへ隠すと、日常利用者の速度が落ちることがある。 このアプリでも「高度を隠す」こと自体は良いが、ONになっているもの、警告があるもの、次に直すべきものは隠さない。
