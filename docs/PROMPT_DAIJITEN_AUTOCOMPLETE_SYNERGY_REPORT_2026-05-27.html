シナジー発見

1. 大辞典 x Prompt Composer Slot

検索候補を単発挿入で終わらせず、Slotテンプレートへ保存すると「モデル別に効く構成」を再利用できる。

実装済み: Slot挿入 次: source別Slot候補

2. 大辞典 x History Tag Review

生成後のaccepted/rejectedタグを大辞典候補と突き合わせると、個人環境で本当に効いた語だけを昇格できる。

履歴レビューあり 次: 成功頻度でrank補正

3. 大辞典 x Model Library

checkpoint/LoRAごとのtrigger、避けるNegative、関連ControlNetを大辞典カテゴリに接続すると、モデル選択がprompt構成まで連動する。

Prompt Profileあり 次: モデル別候補フィルタ

4. 大辞典 x Upscale Pro Recipe

Upscaleで顔崩れや衣装driftが出た時のNegative/補正タグを保存し、次回のAutocomplete上位へ出せる。

Pro Recipe保存あり 次: 失敗理由タグ候補

5. 大辞典 x Tagger

Tagger結果をそのまま使わず、大辞典で正規化・日本語確認・adult marker確認を挟むとノイズを減らせる。

Tagger QAあり 次: Tagger→Dictionary確認面

6. 大辞典 x Reference Board

参照画像の用途ラベルとタグ候補を一緒に保存すると、構図・衣装・色味の再利用が速くなる。

Reference Boardあり 次: 参照ごとのタグメモ

Autocomplete使用感の改善点

  • 候補の2行目に日本語ラベル/意味とpost countを出すことで、同名に近いタグの判断を速くした。
  • 候補右側のbadgeに adult を出すことで、成人語彙を除外せず、挿入前に明示できるようにした。
  • Dictionary画面でも同じ成人/人気度情報を表示し、Autocomplete候補の確認面として使えるようにした。
  • 検索結果型に adultLevel と postCount を追加したため、今後はモデル別rankや履歴rankにも流用できる。

外部ソースと境界

ソース今回の扱い保存境界
Danbooru tags JSON 一般/メタタグの語彙メタデータ。category 0/5、count order、上位ページのみ。 raw画像なし。runtimeへはpromoted tag candidateのみ。
Danbooru posts JSON 前回追加済みの成人向け語彙 evidence。未成年/非合意などは成人語彙として昇格しない。 raw post tag stringはcommitせず、candidate evidenceに集計。
Civitai Red images API 公開画像メタデータのprompt/negativePromptから候補化。成人寄り語もadult marker付きで保持。 画像なし。prompt本文はstaging側、runtimeは昇格候補だけ。

残るリスクと次アクション

  • 意味付け: Danbooru由来の新規語は日本語意味が未レビューのものが多い。意味レビュー/自動curationを次に回す。
  • rank: post countが高いタグは一般的すぎることがある。履歴成功率やモデル別プロファイルでrank補正したい。
  • UI: Autocompleteは候補表示を強化したが、source表示切替やadult表示切替のさらなるrank連動は次スライス。
  • 外部API: Danbooru/Civitaiは外部状態に依存する。再実行時はrate limitとレスポンス変化を確認する。
次の最小スライスは、Dictionary画面に「source / adult / モデル用途」表示切替を置き、Autocomplete rankに History成功タグ、Prompt Composer Slotテンプレート、Checkpoint Prompt Profileを軽く加点すること。
