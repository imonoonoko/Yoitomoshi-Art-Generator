GitHub extension research

Regional Prompter / FABRIC から追加で取り込める改善点

2026-05-13 時点で GitHub の README、主要スクリプト、open issues / PR を確認した。 直近実装済みの最小 UI からさらに伸ばすなら、まずは競合ガードとテンプレート化、次に 2D / Mask / Prompt mode の支援へ進むのが効率的。

調査対象: hako-mikan/sd-webui-regional-prompter, dvruette/sd-webui-fabric

結論

P0

FABRIC と ControlNet reference の同時ONガード

FABRIC README は ControlNet の reference mode と非互換で、同時に有効化すると WebUI 再起動が必要になる可能性を明記している。 アプリ側では ControlNet Builder に「雰囲気参照」があるため、FABRIC 有効時は reference 系 Unit を警告または無効化する。

P1

Regional 2D テンプレート生成

README では ADDCOL/ADDROW と比率からテンプレートを作る導線が重要。 現 UI は単純な BREAK 生成に留まるので、2D グリッド編集とテンプレート反映を追加する。

P1

FABRIC preset と履歴ラベル連携

FABRIC ソースには log/fabric/presets の preset 保存/読込がある。 履歴の「採用候補 / 没」から positive / negative preset を作ると、既存 Library/History と噛み合う。

改善候補一覧

優先 改善案 GitHub で見つけた根拠 実装イメージ
P0 FABRIC と ControlNet reference の競合検知 FABRIC README は WebUI Forge 対応を示しつつ、ControlNet reference mode とは非互換と説明している。 buildAlwaysOnScripts() または UI 側で、FABRIC enabled かつ ControlNet unit の role/model/module が reference 系なら toast 警告。 生成前チェックにして事故を止める。
P0 Regional の BREAK/領域数バリデーション README の Known Issue に、BREAK 数と領域数が合わないと IndexError が起きる旨がある。 Regional UI で ratio から期待領域数を計算し、prompt パーツ数と不一致なら生成前に警告。 useBase/useCommon 有効時の必要パーツ数も表示する。
P1 2D Region Builder README は 2D region assignment と ADDCOL/ADDROW、1,1;2,3,2 形式の比率を説明している。 R1/R2 の一次元リストを、行/列グリッド編集に拡張。 セルの並びから ADDCOL/ADDROW テンプレートと divide ratio を自動生成する。
P1 Regional 可視化プレビュー README は ratio を入力して領域を可視化し、prompt template を作る操作を紹介している。 Canvas/SVG ではなく React の軽量 CSS grid で十分。 ratio に応じたセルサイズ、Rows/Columns/flip の反映、セル別 prompt の省略表示を作る。
P1 Mask mode の入口 README と open PR #413 は Mask mode の uploaded mask、色一致、ADDCOL/ADDROW クラッシュ回避に触れている。 最初は手描きエディタではなく、既存の inpaint mask / 外部 mask 画像を Regional の polymask へ渡す限定実装にする。 PR #413 の未マージ状態を踏まえ、mask mode は検証付き beta として表示する。
P1 FABRIC preset 保存/読込 FABRIC ソースは log/fabric/presets に liked/disliked paths の JSON preset を保存する関数を持つ。 positive/negative リストを preset 名で保存し、Workspace とは別に再利用できる UI を追加。 履歴ラベル「採用候補」「没」から preset 生成するショートカットも候補。
P2 FABRIC feedback 枚数と強度のプリセット README は feedback 枚数が多いとモデルが混乱する場合があり、強度を下げるのが有効と説明している。 1-2枚、3-5枚、6枚以上で推奨 max weight を提示。 画像枚数が多い場合は maxWeight と end の推奨を UI に出す。
P2 Regional Latent mode の慎重な露出 README は Latent mode をサポートするが遅く、Forge/reForge では制約があると説明。 open issue #418 では latent option 使用時のエラー報告もある。 UI に Latent を出す場合は「実験的」「Forge で要検証」と表示し、既定は Attention のままにする。 まずは payload だけでなく実生成比較を作ってから正式機能化する。
P2 Forge / Forge Neo 互換診断 Regional issue #414/#410、FABRIC issue #40 は Forge 系互換の問題や要望を示している。 起動診断に拡張ファイル存在、主要 import、Forge variant を表示。 壊れている拡張を有効化しないための health check を Tools に追加する。

次の実装順

  1. 競合ガード: FABRIC ON + ControlNet reference 系 Unit を検知して生成前に止める。
  2. Regional バリデーション: ratio、use common、use base、region prompt 数の整合性を UI で表示する。
  3. 2D テンプレート: ADDCOL/ADDROW を生成する小さなグリッド UI を追加する。
  4. FABRIC preset: positive/negative の保存/読込と、履歴ラベルからの投入を作る。
  5. 実生成比較: Regional 1D/2D、FABRIC ON/OFF、ControlNet reference 競合回避の比較を同一 seed で残す。
