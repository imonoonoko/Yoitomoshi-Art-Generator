AIキャラ追加を「置く・馴染ませる・記録する」まで拡張

前回のP0では、元画像へキャラを置き、Inpaint maskとControlNet Unit 1を作るところまでを実装した。 今回は実運用で必要になる、背景トーン合わせ、キャラ参照のControlNet Unit 2自動投入、Before/After保存レポートまで追加した。

Before / After

Before
  • 元画像とキャラ画像を合成し、Inpaint maskとControlNet Unit 1を作成。
  • キャラの色調はプリセット固定で、背景ごとの明るさ差は手動調整が必要。
  • 生成後の比較はユーザーが履歴やスクリーンショットで別管理する必要があった。
  • IP-Adapter / reference_only 系の参照維持は手動でControlNet Unitを追加する必要があった。
After
  • キャラ周辺の背景トーンを測り、合成時に明るさ・彩度・コントラストを自動補正。
  • Reference/IP-Adapter系ControlNet moduleとmodelがある場合、キャラ画像をUnit 2へ自動投入。
  • 元画像、キャラ、仮配置、mask、生成後画像を1フォルダへ保存し、HTMLレポートを生成。
  • 一度「馴染ませ準備」しても元画像をローカル保持し、再調整時に合成が二重にならないようにした。

保存される比較パッケージ

画像

base.png, character.png, placed-composite.png, inpaint-mask.png, 必要に応じて generated-after.png を保存。

Manifest

prompt、negative、preset、denoise、ControlNet module/model、位置/サイズ/回転、mask設定を character-composite.json に保存。

HTML

character-composite-report.html で、元画像から生成後までを横断的に確認できる。

狙い

  1. LayerDiffuseで透明キャラを作れる環境が整ったら、そのPNGを即このパネルへ送れる。
  2. IP-Adapter / reference_onlyがある環境では、キャラの顔や服を崩しにくくする。
  3. IC-Lightを入れる前でも、最低限の背景トーン合わせで「貼り付け感」を減らす。
  4. 採用/没の判断材料をHTMLとmanifestに残し、後から同じ素材で再試行できる。

次に残る実素材QA

  • 同一写真で「自動トーンON/OFF」を比較し、暗所写真で不自然に沈みすぎない範囲を確認する。
  • IP-Adapter modelを配置した環境で、Unit 2のmodel選択と顔・服保持の効きを確認する。
  • LayerDiffuse導入後、透明PNGの生成結果をこのパネルへ直接渡す導線を追加する。
  • 保存HTMLのgenerated-after.png付きケースを実生成後に確認する。
