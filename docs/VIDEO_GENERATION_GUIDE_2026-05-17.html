まず覚える場所

動画生成でもベースになるStable Diffusion checkpointは使います。motion moduleはそのcheckpointに時間方向の動きを足す部品です。 motion module自体はcheckpoint置き場ではなく、AnimateDiff拡張の model フォルダへ置きます。

C:\宵灯工房アート\Yoitomoshi-Art-Generator\runtime\forge\webui\extensions\sd-forge-animatediff\model

1. モデルをダウンロードする

初回はSD1.5系checkpointと mm_sd15_v2.safetensors の組み合わせが一番わかりやすいです。 SDXL checkpointにはSD1.5用motion moduleを使えません。SDXL系を使う場合はSDXL対応のmotion moduleを別途選びます。

用途 ダウンロードするファイル 目安サイズ 最初の設定
初回の動作確認 mm_sd15_v2.safetensors 約909 MB SD1.5 checkpoint / 8 frames / 8 FPS / Context batch 4
SD1.5の別候補 mm_sd15_v3.safetensors 約837 MB v2で動作確認後に試す。まずは8 framesから。
SDXLを試す場合 mm_sdxl_hs.safetensors 約475 MB SDXL checkpoint専用。HotShot-XLはContext batch 8推奨。
SDXLの別候補 mm_sdxl_v10_beta.safetensors 約475 MB SDXL checkpoint専用。初回はSD1.5で通してから。

混ぜない

SD1.5 checkpointにはSD1.5用motion module、SDXL checkpointにはSDXL用motion moduleを使います。 例: n4mik4ILSFWNSFW_v20 のようなSDXL checkpointに mm_sd15_v2.safetensors を合わせると失敗します。

ブラウザで落とす場合

  1. 上の mm_sd15_v2.safetensors リンクを開く。
  2. ダウンロード完了を待つ。
  3. ファイル名を変えずに model フォルダへ移動する。

PowerShellで落とす場合

回線やHugging Face側の応答で時間がかかるため、途中で止まる場合はブラウザダウンロードへ切り替えます。

$dest = "C:\宵灯工房アート\Yoitomoshi-Art-Generator\runtime\forge\webui\extensions\sd-forge-animatediff\model"
New-Item -ItemType Directory -Force $dest
Invoke-WebRequest `
  -Uri "https://huggingface.co/conrevo/AnimateDiff-A1111/resolve/main/motion_module/mm_sd15_v2.safetensors?download=true" `
  -OutFile (Join-Path $dest "mm_sd15_v2.safetensors")

2. motion moduleを配置する

1

アプリからフォルダを開く

Video タブを開き、Motion module置き場のフォルダボタンを押す。

2

ファイルを置く

mm_sd15_v2.safetensors をそのフォルダへ入れる。サブフォルダに入れず直置きでよい。

3

再確認する

Video タブの 再確認 を押す。候補欄にファイル名が出れば配置は通っている。

Forgeの再起動

拡張を追加した直後や、生成時にAnimateDiffが効いていない場合は、Forgeを一度停止して再起動します。 motion moduleファイルの追加だけなら 再確認 で見えることがありますが、初回は再起動した方が判断しやすいです。

3. Videoタブで生成する

txt2vidの最短手順

  1. Forgeを ready にする。
  2. 上部の Video タブを開く。
  3. 動画ベース でベースcheckpointを選ぶ。
  4. ソース切替で txt2vid を選ぶ。
  5. prompt / negative promptを書く。
  6. Motion moduleに mm_sd15_v2.safetensors を選ぶ。
  7. 動画生成スイッチをONにする。
  8. 短尺設定で 動画を生成 を押す。

img2vidの最短手順

  1. Video タブを開く。
  2. ソース切替で img2vid を選ぶ。
  3. 入力画像欄に画像を入れる。
  4. promptは画像をどう動かしたいかに寄せる。
  5. denoiseは低めから試す。まずは 0.35 から 0.55。
  6. 生成後は userdata\videos に保存される。

初回スモーク設定

項目 推奨値 理由
checkpoint SD1.5系 mm_sd15_v2.safetensors と合わせやすい。
解像度 512 x 512 または 512 x 768 最初はVRAMと時間を抑えて、成功/失敗を早く見る。
Steps / CFG 16 steps / CFG 6 短尺確認では画質より通ることを優先する。
Frames / FPS 8 frames / 8 FPS まずは約1秒でAnimateDiffが効くかだけを見る。
Context batch 4 8GB前後のVRAMでも通しやすい初回値。成功後に8-16へ上げる。
Format GIF 最初の確認がしやすい。あとでMP4/WEBP/WEBMへ切り替える。
Batch / Iterations 1 / 1 Video生成では複数枚同時生成を避ける。

最初から高解像度にしない

動画生成は静止画よりVRAMと時間を使います。いきなり1024px超や長尺にせず、 まず 512 x 512 / 8 frames / Context batch 4 / GIF でAnimateDiffが動くことを確認します。
