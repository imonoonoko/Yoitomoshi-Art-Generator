Before / After Report

ControlNet Builder 実装ビフォーアフター

ControlNet入力生成の導線を、画像ドロップ、preprocessor実行、プレビュー、ControlNet Unit反映まで一続きで扱える形へ強化した。 既存のUnit 1固定を外し、選択Unitへ反映できるようにした。

関連レポート: 拡張機能シナジー調査

1. 実装前後

Before

構図プリセットは用途選択、画像ドロップ、preprocessor、プレビュー、Unit 1反映に対応していた。

反映先はUnit 1固定 preprocessor実行と反映が別操作 既存ControlNet payloadは維持

After

反映先Unitを選び、必要ならUnitを追加し、preprocessor実行からUnit反映までをワンボタンで完了できる。

Unit選択 Unit追加 生成して反映 プレビュー単体反映も維持

2. 変更内容

ファイル変更狙い
src/components/extensions/ControlNetBuilderPanel.tsx 反映先Unit selector、Unit追加、runAndApply()、Unit指定のapply処理を追加。 ControlNet入力生成を「素材を入れる、用途を選ぶ、生成して反映する」という短い作業にまとめる。
src/lib/i18n.ts Unit選択、生成して反映、Unit指定toastをja/en/ru/ptへ追加。 多言語UIで未翻訳キーが露出しないようにする。

3. 操作フロー

1. 入力

参照画像をドロップまたはクリック選択し、用途プリセットと反映先Unitを選ぶ。

2. 生成

生成して反映でpreprocessorを実行し、必要な画像マップを作る。

3. 反映

生成結果、module、model、weight、control modeを選択Unitへ反映する。従来の手動プレビュー反映も残す。

4. 検証

項目結果
npm.cmd run typecheck PASS ControlNet BuilderのUnit選択、非同期preprocessor、i18n参照に型エラーなし。
npm.cmd run build PASS Electron/Viteの本番ビルドに成功。
Electron DOM確認 PASS Unit 1〜3の選択、テスト画像投入後の生成して反映表示、プレビュー枠表示を確認。
互換性 維持 既存のControlNet stateとpayload形式は変更せず、反映先indexだけを選べるようにした。
残る確認 実生成確認推奨 Forge側のpreprocessor実行時間と生成結果は、実画像を使って手動確認する。
