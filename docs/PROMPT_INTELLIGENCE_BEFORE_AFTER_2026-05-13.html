Before / After Report

Prompt Intelligence 実装ビフォーアフター

拡張機能調査レポートのP0「Prompt Intelligence Hub」に対する実装スライス。 Better Prompt / Prompt All in ONEで重要な「タグの並び替え」と「複数タグの一括操作」を、既存のprompt文字列保存方式を維持したまま追加した。

参照レポート: EXTENSION_GITHUB_SYNERGY_REPORT_2026-05-13.html

1. 実装前後

Before

PromptPanelのタグチップは表示、重み調整、削除、positive/negative間移動、重複整理まで対応していた。

タグ順序はtextarea手編集 複数タグ操作は1件ずつ よく使う部分タグ列の保存が遠い 構図/品質/LoRA triggerの優先順位整理が手間 既存prompt文字列は正本として維持

After

タグチップをドラッグで並び替え、チェックした複数タグをまとめて重み変更・削除・positive/negative移動・QuickPreset保存できる。

ドラッグ並び替え 複数選択 一括重み変更 一括移動/削除 選択タグ列をQuickPreset保存 日本語/英語ラベル併記維持 positive/negative両対応 保存形式変更なし

2. 変更内容

ファイル 変更 狙い
src/lib/prompt-utils.ts reorderPromptToken()、removePromptTokensByIndexes()、adjustPromptTokensByIndexes() を追加。 カンマ区切りpromptをタグ配列として扱い、並び替え、一括削除、一括重み調整を安全に再結合する。
src/components/PromptTagChips.tsx drag/drop、選択チェック、一括操作バー、選択タグ列のQuickPreset保存を追加。 Prompt All in ONE / Better Prompt系のタグ単位編集体験と、Easy Prompt Selector系の再利用導線を接続する。
src/components/PromptPanel.tsx 単体移動callbackを複数タグ移動callbackへ拡張。 選択タグをpositive/negative間で一括移動しても、既存のpromptAppend()重複回避を使う。
src/lib/i18n.ts drag/drop、複数選択、一括操作、選択保存の文言をja/en/ru/ptへ追加。 多言語UIの欠落を避ける。

3. 操作フロー

1. タグを作る

Prompt Library、QuickPreset、英語タグ変換、手入力でpromptを作る。

2. チップを整える

タグチップをドラッグして順序を調整し、チェックしたタグをまとめて重み変更、削除、positive/negative移動する。

3. 再利用して生成へ渡す

よく使うタグ列はQuickPresetへ保存でき、編集結果はtextareaのprompt文字列に戻るため履歴、Workspace、生成payloadは従来どおり動く。

実装方針: 新しい構造化prompt stateはまだ導入していない。既存の保存・履歴・Workspace互換を守るため、prompt文字列を正本にしたまま操作UIだけを強化した。

4. 検証

項目結果
npm.cmd run typecheck PASS React drag/drop型、複数選択イベント型、QuickPreset保存API、i18n参照、prompt-utils exportに問題なし。
Electron DOM確認 PASS remote debugging経由でテストpromptを投入し、5件のタグチップ生成、2件選択、一括操作バー表示を確認。
保存互換 維持 prompt/negativePromptはstringのまま。Workspace/History/Generation payloadの形式変更なし。
残る確認 手動UI確認推奨 スクリーンショット取得はCDP応答が返らず未保存。Electron実画面でドラッグ、複数選択、QuickPreset保存の手触りを確認する。

5. 次の改善候補

  • タグ無効化: 文字列promptだけでは安全な無効化表現が難しいため、構造化Prompt Recipeを導入する段階で実装する。
  • My Prompt拡張: 今回はQuickPreset保存に寄せた。次はPrompt Libraryのユーザーカテゴリや補完候補にも露出する。
  • Blacklist: ネガティブへ送るべきタグ、重複しやすいタグ、LoRA triggerと競合するタグを検出する。
  • 構造化Prompt Recipe: タグ順序、無効化、グループ名、プリセット由来を保持し、文字列promptへエクスポートする層を追加する。
