結論

1. プロンプトは「文章」より「制作指示書」

Stable Diffusion系では、主題、構図、視線、動作、衣装、背景、光、品質タグ、除外タグを分けて扱う方が再利用しやすい。 Midjourneyの公式ドキュメントも、短く明確な言葉と、最後に置くパラメータの分離を重視している。

2. 設定はモデル別に固定し、seedで比較

CFG、steps、sampler、size、denoiseは「神設定」として単独で効くものではなく、モデル、LoRA、ControlNet、Hires/Upscaleと結びついて効く。 アプリ側は固定seedのA/B比較を最短操作にするべき。

3. 拡張は制作工程に割り当てる

ControlNetは構図、ADetailerは顔や手の補修、Regional Prompterは領域分離、IP-Adapterは参照保持、Ultimate/Tiledは高解像度化。 名前で並べるより「何を直すか」でUIに出す方が初心者向け。

プロンプト作法

作法 実務での意味 Yoitomoshiへの落とし込み
短く明確 Midjourneyは短く単純なプロンプトを推奨し、主題、媒体、環境、照明、色、ムード、構図を意識するよう案内している。 SDでも長いタグ列をただ足すより、重要度の高い要素を残す方が安定する。 Prompt Helperに「主題 / 行動 / 場所 / 光 / 構図 / 仕上げ」のスロットを作り、 未入力スロットだけを提案する。全文置換ではなく既存プロンプトへ追加する。
重みは少数だけ A1111は `(word:1.5)` などの明示重みを持つ。便利だが、全部を強調すると優先順位が消える。 強調するのは顔、衣装のキー要素、光、避けたい崩れのような少数に絞る。 タグチップに 0.8 / 1.0 / 1.2 / 1.4 の軽い重みボタンを追加する。 1.6以上は「強すぎる可能性」表示にする。
ネガティブは目的別 Diffusersの公式説明では、negative promptは望まない内容や低品質特徴から離すためのもの。 汎用ネガティブを巨大化させるより、手、顔、文字、透かし、余計な物など目的別に分ける。 Negative Promptを「品質」「人体」「文字/透かし」「不要物」「スタイル抑制」のプリセットに分け、 生成目的に応じてON/OFFできるようにする。
モデルカード優先 Animagine XL 4.0のモデルカードは、タグ順、品質タグ、推奨negative、CFG 4-7、steps 25-28、Euler aを明記している。 NoobAI、Pony、Illustrious系もモデルごとの作法が強いので、一般設定で上書きしない。 Model Libraryへ「prompt contract」を追加する。 checkpointごとに開始タグ、末尾品質タグ、推奨CFG/steps/size、避けるタグを持たせる。
LoRAは効果量を管理 A1111のLoRA記法は `` で、基本的に0から1の強さを使う。 衣装LoRA、画風LoRA、detail LoRAを積むほど競合しやすい。 LoRAカードに「用途: キャラ / 衣装 / 画風 / detail」と推奨weightを保存し、 1.0超や複数detail LoRAの同時使用を警告する。

神設定の実体

固定の万能値は作らない。アプリの既定値は「まず破綻しにくい値」にし、神設定は モデル別プリセットとA/B比較のセットとして扱う。

設定 推奨初期値 調整ルール UI化
CFG / guidance SDXL anime: 5前後 Diffusersは高すぎるguidanceでartifactが出る可能性を説明している。 効かない時はCFGを上げる前に、競合タグ削除、LoRA weight調整、ControlNet追加を試す。 4 / 5 / 6 / 7 の比較ボタン。7超は警告。
Steps 20-30 単純な顔/立ち絵は20前後、背景やアクションは25-30。 stepsを増やすより、構図をControlNetで固定した方が効くケースが多い。 「軽量」「標準」「精密」の3択。
Resolution SDXL: 1024基準、8の倍数 DiffusersはSD1.5の標準512、SDXLの標準1024を明記している。 8GB VRAMでは長辺1536以上は直接生成よりUpscaleへ回す。 縦長/横長/正方形の安全サイズプリセット。
Hires / Upscale denoise 0.25-0.35 Ultimate SD Upscaleの例はdenoise 0.35-0.4付近を示すが、 本アプリではtile drift対策として人物素材は0.25を基準にする。 「保持」「標準」「再描画」の3段階。人物は保持を既定。
Seed 比較時のみ固定 Diffusersはgenerator/seedで再現性を確保できる。 設定比較、LoRA weight比較、ControlNet ON/OFF比較では必ず固定する。 比較生成に入る時だけ自動固定し、終了時に解除確認。

制作ワークフロー

キャラクター立ち絵

  • Checkpointのprompt contractを適用。
  • キャラ/髪/表情/服/ポーズ/背景の順でタグを組む。
  • LoRAはキャラ1、衣装1、画風1までを初期上限にする。
  • ADetailerで顔、必要なら手を補修。
  • 採用候補をHistory labelへ保存。

一枚絵・背景込み

  • 最初に構図タグとカメラ距離を決める。
  • OpenPose/Depth/Lineartのどれで固定するか選ぶ。
  • Regional Prompterは前景/背景/小物の分離に使う。
  • UpscaleはTile ON / denoise 0.25を基準に比較。

既存絵から再構成

  • Taggerで画像タグを抽出し、Prompt Helperへ渡す。
  • IP-Adapterまたはreference系ControlNetで雰囲気を保持。
  • 強く寄せたい時はControlNet、緩く寄せたい時はimg2img denoiseを下げる。
  • 著作権/キャラ名タグは用途に応じて明示的に除外できるようにする。

素材化・ゲーム向け出力

  • LayerDiffuseや背景除去系は「素材化済み」ラベルとつなげる。
  • 透過素材、背景差分、表情差分をWorkspaceへ参照保存。
  • 同じseedとprompt contractで表情/ポーズだけ差分生成する。
  • 最終出力はHistoryからUpscaleへ送って保存。

必須級拡張と採用判断

優先 拡張 / 技術 何に効くか 採用メモ
P0 ControlNet ポーズ、線画、奥行き、構図、参照画像の制御。 既にControlNet Builderがあるので、次は用途別カードを 「ポーズ固定」「線画化」「奥行き」「参照雰囲気」に整理する。
P0 ADetailer 顔、手、目、小物などの自動検出とinpaint補修。 本アプリではdicts-only payloadを維持する。 UIは「顔を整える」「手を整える」の簡単な目的名で出す。
P0 Tagger 入力画像からタグを抽出し、Prompt Helperへ戻す。 WD系を安定既定にし、PixAI/Camieなど新規taggerはライセンスと実行形式を分けて評価する。 商用配布に注意が必要なものは同梱しない。
P0 Ultimate SD Upscale / Tile ControlNet 高解像度化、細部追加、VRAM制限下の分割処理。 現在のUpscale比較と相性が良い。 既定は人物保持寄り、detail追加は明示的な比較候補にする。
P1 IP-Adapter 参照画像の雰囲気、顔、服、構図をプロンプト以外で保持。 ControlNet Unit 2やCharacter Compositeに接続する価値が高い。 非正方形画像は情報欠落が起き得るので、参照クロップUIが必要。
P1 Regional Prompter 領域ごとに髪色、服、背景、小物などを分ける。 `BREAK`、`ADDROW`、`ADDCOL`を直接書かせるより、 画面分割テンプレートからpromptを生成する方が扱いやすい。
P1 LayerDiffuse / IC-Light 透過素材、背景合成、照明合わせ。 ゲーム素材化に直結するが、まずは導入診断と不足モデル表示を優先する。
P1 Dynamic Thresholding / FreeU 高CFG時の色破綻抑制、U-Net出力の質感改善。 初心者向け既定にはせず、上級設定またはプリセット内でON/OFF比較する。
保留 ComfyUI巨大workflow移植 ノードベースの柔軟な制作。 ComfyUIはノード型で強力だが、このアプリはForge API専用起動が前提。 workflow思想だけを取り込み、ComfyUI依存は増やさない。

Yoitomoshiへの実装提案

P0: Prompt Contract

Model Libraryにモデル別の推奨タグ順、品質タグ、negative、CFG、steps、size、samplerを保存する。 既存のHugging Face検索、Civitai metadata、safetensors inspectと自然に接続できる。

P0: Compare Studio

seed固定でCFG、steps、LoRA weight、denoise、ControlNet ON/OFFを少数候補で横並び生成する。 既存のUpscale比較保存をtxt2img/img2imgにも広げる。

P0: Prompt Helper 2

自然文補助ではなく、主題、動作、構図、照明、質感、negative、LoRA、ControlNetのチェックリストにする。 入力済みプロンプトを壊さず、足りない要素だけ提案する。

P1: Extension Recipe Cards

「顔を直す」「手を直す」「ポーズ固定」「線画化」「背景と前景を分ける」「高解像度化」のように、 拡張名ではなく制作目的カードから設定を作る。

P1: Regional Template Builder

1x2、1x3、2x2、前景/背景、人物/小物/背景のテンプレートを作り、 `BREAK`や`ADDROW`をUIが自動生成する。

P1: Source Reliability Badge

プリセットの出典を「モデルカード」「公式拡張README」「ユーザー検証」「コミュニティ記事」に分ける。 コミュニティ由来の神設定は必ず固定seed比較を要求する。

注意点

  • コミュニティの神設定は、checkpoint、VAE、LoRA、sampler、scheduler、clip skip、解像度が揃わないと再現しない。
  • AGPL/GPL/CC BY-NC-SA系の拡張やモデルは、同梱、再配布、商用利用の扱いを分ける。
  • Forge API専用運用では、Gradio UI前提の拡張をそのまま使えないことがある。alwayson_scriptsの引数順を固定し、変更検知を入れる。
  • 画像参照、tagger、IP-Adapterは便利だが、著作権キャラ名やartist tagの混入を避けるフィルタが必要。
