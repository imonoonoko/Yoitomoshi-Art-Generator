Executive Summary

PixAI は公式 Platform API と公開Web GraphQLの両面から、モデルID、モデルバージョン、モデル別LoRA推薦、サンプラー候補を調べられる。 Aipictors は公開GraphQLとRemix SSRデータから、搭載モデル一覧、人気タグ、公開作品の生成設定を取得できる。 ただし、どちらも「外部サイトの生成・投稿・お気に入り・ログイン情報」をYoitomoshiから操作する方向は避け、まずはローカル制作を補助する 読み取り専用のトレンド辞書・レシピ取り込み・設定比較に落とすのが安全。

PixAI公式API採用候補

APIキー前提。生成タスクやmodelVersionId中心の連携は公式経路を優先。

PixAI Web GraphQL低〜中信頼

公開read-only queryは動くが、未公開契約なのでトレンド調査・手動確認向け。

Aipictors公開GraphQL中信頼

搭載モデル一覧と初期設定は取得可能。生成・投稿系は対象外。

Yoitomoshi実装有用

外部トレンドをローカルモデル・LoRA・Prompt Helperへ変換する補助機能が向く。

Current Findings

PixAI

  • 公式Helpは、Model/LoRA market、Variation、Use as Base Image、Enhance、Note、ControlNet、Composition、Prompt Helper相当の導線を説明している。
  • 公式Platform APIは modelVersionId を中心にした v2 image create を案内しており、モデルページURLの最後のpath segmentをIDとして使う。
  • 公開Web GraphQLでは、モデル一覧・モデルversion・model別LoRA推薦・類似LoRA・公開artwork task detailのoperationが確認できた。
  • 実測例: Anime Illustration Diffusion のモデルID 1667980440770052992 / version 1667980443970306972 に対し、推薦LoRAとして DMD2、Hyper SDXL、Hyper SDXL 2steps-lora などが返った。
  • 公式記事の設定指針は、探索ではsteps 15-20 / CFG 5、開発ではsteps 28、仕上げではsteps 35-40、CFGは概ね5前後を推奨。HelpではCFGは7未満を推奨している。

Aipictors

  • /models から搭載モデル77件を確認。例: ChromaFTv4、ChromaXLv1b、flux.1 schnell、flux.1 pro、zukiAnimeILL v5.0、animagineXL4.0 Opt、Visionary Illustrious Mix、PuellulaXL-1536、4nimaPencilXL。
  • 公開configから、default promptは masterpiece, best quality, extremely detailed, anime, girl, skirt、default negativeは EasyNegative, bad_prompt_version2, badhandv4。
  • 公開configの既定値は steps 20、sampler DPM++ 2M Karras、scale 7、clip skip 2、VAE vae-ft-mse-840000-ema-pruned。steps範囲は9〜30として定義されている。
  • 公開configのdefault LoRA名は flat1 / flatBG。無料・liteではLoRA 2件、standard/premiumでは5件までの設定値が見える。
  • /search の人気タグ例: ホロライブ、美少年、メイク、パーカー、ワイン、ゴルフ、初音ミク、ぐっすり、カチューシャ、成人女性。
  • 公開ランキング作品例では、prompt / negative / seed / steps / sampler / scale / dailyRanking / weeklyRanking が公開データとして埋め込まれていた。prompt非公開作品は対象外にするべき。

Recommended Implementation Order

  1. P0: 設定プリセット辞書をローカル実装する。外部通信なしで、PixAI/Aipictorsの公開設定知識をPrompt HelperとPreflightへ入れる。
  2. P0: URL手動取り込みを追加する。ユーザーが貼ったAipictors公開postだけを解析し、Recipe候補を作る。
  3. P1: External Trend CacheをToolsに追加する。手動更新、出典表示、キャッシュ期限、失敗時の無効化を入れる。
  4. P1: PixAI推薦LoRAのCivitai照合を試す。PixAIの推薦をそのまま使わず、Civitai/localに一致したものだけ適用候補にする。
  5. P2以降で、許可アカウント・redaction方針を確認してから、PixAI/Aipictorsのログイン後flowをcaptureする。

Open Questions Before Deeper Capture

  • PixAI / Aipictors のログイン済みflowを調べる場合、どのアカウントを使うか。自分のアカウント・自分の作品・自分の生成履歴だけに限定する必要がある。
  • 保存/変更操作を許可するか。生成、投稿、favorite、bookmark、publish、credit消費は明示許可があるまで実行しない。
  • Raw traceを保存してよいか。保存する場合はcookie、authorization、csrf、email、user id、task id、非公開promptをredactする。
  • Yoitomoshiに外部APIキーを保存するか。保存するなら暗号化・portable userdata・削除導線が必要。
