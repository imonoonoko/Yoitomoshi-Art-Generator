3. 主要ワークフロー

txt2img生成

Prompt/Negative/Params/Extensionsを編集する。
Preflightで不足や危険設定を確認する。
buildGenerationPlanがForge requestを組み立てる。
forge.txt2imgで生成し、画像とthumbnailを履歴へ保存する。
LoRA使用ログを保存し、次回提案に使う。

画像投入と再利用

D&DまたはClipboardで画像を投入する。
PNG/JPEG/WebP metadataをbest-effortで解析する。
設定を復元し、Civitai照合をバックグラウンドで実行する。
img2img、Upscale、Variation、Recipe保存へ分岐する。

Upscale品質判断

画像寸法とmetadataから推奨設定を出す。
Simple、Diffusion、Ultimateから方式を選ぶ。
必要ならUpscale専用Tile ControlNetを有効にする。
denoise 0.25/0.35/0.45とTile ON/OFFを比較する。
判断基準と候補画像を保存する。

History review -> Prompt Helper

履歴画像をTagger reviewで開く。
PixAI ONNXの候補をaccepted/rejectedに整理する。
userdata/history/index.jsonへ保存する。
Prompt Helperや比較スクリプトが保存済みレビューを再利用する。

4. データの置き場

場所 Git 役割 注意
src/, electron/ 管理する Renderer、main、IPC、Storage、Forge連携の実装。 既存の責務境界に合わせて小さく変更する。
resources/ 管理する 同梱プロンプトライブラリ。 YAML変更時はparse検証を行う。
docs/ 管理する ロードマップ、QA、構造、調査、今回の可視化成果物。 コード変更後は実装状況とズレないよう更新する。
runtime/ 管理しない Forge本体、モデル、拡張、Python/Git。 巨大で環境依存。原則Gitに入れない。
userdata/ 管理しない 設定、秘密情報、履歴、Civitai cache、downloads、workspaces。 実データやAPIキーを含む。QA fixtureは必ず戻す。

5. 壊してはいけない契約

ADetailer payload

dicts-only形式を維持する。bool配列形式へ戻すと拡張側の引数解釈が壊れやすい。

Ultimate SD upscale args

script_args18個の順序は拡張のui()戻り順に依存する。並べ替えない。

Upscale専用ControlNet

通常ControlNetとTile ControlNetを混ぜない。forUpscaleDiffusionなどの専用contextを維持する。

4タブ方針

トップレベルはtxt2img/img2img/Upscale/Tools。新機能は既存タブ内の折りたたみパネルへ入れる。

DOM QAセレクタ

表示文言ではなくdata-testidと状態属性を使う。ja/en/ru/ptで壊れないようにする。

生成物と実データ

runtime/、userdata/、output/、out/、dist/は原則Gitに入れない。
