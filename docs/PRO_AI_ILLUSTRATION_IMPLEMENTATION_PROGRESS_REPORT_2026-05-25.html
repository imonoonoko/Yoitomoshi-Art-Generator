制作ループの変化

今回の実装は、単にPromptを長くする機能ではなく、成功条件を保存して次の制作判断へ戻す流れを作ったものです。

1. モデル作法を読む Checkpoint Profile Pro が baseModel、Prompt形式、Negative方針、推奨比率、LoRA数を提示。
2. 仕様書からPrompt化 Prompt Composer Slots が主題、構図、光、色、背景、破綻回避をモデル向けに整形。
3. batch候補を選別 Candidate Board で seed、index、ラベル、送信先を同じ面で操作。
4. Pro Recipe保存 Historyへ rating、強み、弱点、次アクションを保存し、偶然の成功を再利用可能に。
5. Civitai傾向を戻す Model Libraryが trigger words、recommended prompts、community傾向を制作UIへ渡す。

実装フェーズ

Phase 1からPhase 5までを実装済み。Phase 6は設計確認のみで、次の任意フェーズとして残しています。

P1

Pro Recipe Review MVP

Historyに制作判断を保存

  • HistoryProRecipeReview と保存IPCを追加。
  • rating、strengths、issues、nextActionsをHistory GUIから編集。
  • tagReview と責務を分離し、既存履歴の後方互換を維持。
P2

Model Profile Pro

モデルごとのPrompt作法と推奨設定

  • baseModel、Prompt形式、Negative方針、推奨比率、LoRA数を追加。
  • Preflightがモデル推奨と現在設定の差分を表示。
  • Model Library内にコンパクトなプロファイル編集面を追加。
P3

Pro Prompt Composer Slots

視覚仕様書からPromptを生成

  • 品質/Score、主題、構図、表情/ポーズ、光、色、服/小物、背景、質感、仕上げ、破綻回避slotを追加。
  • タグ型、自然文型、構造化型をCheckpoint Profileに連動。
  • 品質/Scoreと仕上げを分離し、Ponyは品質先頭、Animagineは品質末尾、Fluxは自然文寄りのプロ順に整理。
  • Libraryクリックをslotへ入れるON/OFFを追加。
  • 破綻回避slotからNegative Promptへ反映。
P4

Candidate Board

batch候補の選別と次工程への送信

  • 最新batchをHistoryから抽出し、候補グリッドとして表示。
  • favorite、candidate、rejected を候補カード上で保存。
  • 候補画像クリックで中央Previewへfull image表示。
  • 候補画像をimg2img、Upscale、Pro Recipeへ送信。
P5

Civitai Recipe / Trend Importer

流通レシピをModel LibraryとPrompt Composerへ

  • trainedWords と recommendedPrompts をModel Library metadataへ追加。
  • LoRA系のCivitai更新を fetchLoraByHash() に接続。
  • community statsからSampler、Steps、CFG、Size、LoRA、共通phraseを表示。
P6

Reference Board / Optional Backends

次フェーズ。今回は設計確認のみ

  • 参照画像slot、ControlNet、Workspace image referenceの既存契約を確認。
  • 外部backendはForge標準生成payloadへ混ぜない方針を維持。
  • 実装はPhase 1-5の制作ループ安定後に回す。
