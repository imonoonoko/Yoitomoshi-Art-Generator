実装内容

各カードは「どの画面で効くか」と「何が保存・検証されるか」を中心に整理しています。

Tools

起動と復旧の安定化

個人環境ヘルスチェックを追加し、復旧と診断をToolsへまとめました。

  • stale running DownloadJob、壊れたsettings、残プロセス、orphan partialを診断。
  • 自動復旧はsettings正規化、stale DownloadJob整理、Model Library復旧に限定。
  • 残プロセス停止や孤立partial削除は手動確認対象として扱う。
History

制作履歴の検索性

履歴をサムネ一覧から、検索できる制作資産へ寄せました。

  • prompt、negative、model、sampler、LoRA、tagReview、rating、Pro Recipe、labelを横断検索。
  • 成功、favorite、没、素材用、SNS向け、参考用、Pro Recipeありを素早く抽出。
  • 一覧上に「何が良かったか」「次に何をするか」のメモを表示。
Models

Download / Model Library後始末

モデル管理の曖昧な残骸を、安全に見分けて片付けられるようにしました。

  • running、stale、failed、orphan partialをModel Library UIで分離。
  • 実行中downloadの破棄/再開をmain側で拒否し、stale化したものだけ対象化。
  • checkpoint profileへLoRA / VAE / ControlNetの対応関係を集約。
Prompt

Prompt資産の育成

よく使う構成を、毎回の手入力ではなく育てるテンプレートへ変えました。

  • Prompt Composer Slotテンプレートを保存、読込、削除可能に。
  • モデル別に効く構成、避けるNegative、推奨LoRA数をCheckpoint Prompt Profileへ保存。
  • タグ辞書は用途別レシピとして、SlotまたはPrompt/Negativeへ挿入。
Board

Candidate Board強化

候補比較を「見比べる場」から「判断を残して派生を作る場」へ拡張しました。

  • 採用理由、失敗理由、次アクションをPro Recipe reviewへ保存。
  • seed+1、CFG +/-0.5、LoRA weight +/-0.05をtxt2imgへ直接反映。
  • SNS向け、素材用、参考用など用途ラベルで後から探せる。
Reference / Upscale

参照と仕上げの接続

参照素材とUpscale判断を、WorkspaceとPro Recipeに残る形へまとめました。

  • Reference Boardで参照画像、ポーズ、色味、キャラ資料、由来メモを管理。
  • img2img、Inpaint準備、ControlNet Unit 1への導線を統一。
  • Upscaleの顔崩れ、衣装drift、線の破綻などをチェックし、採用設定を自動保存。

Extensions

API readiness
