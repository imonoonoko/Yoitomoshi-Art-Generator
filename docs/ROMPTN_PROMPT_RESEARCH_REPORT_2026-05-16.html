Research Report

Romptn Stable Diffusion プロンプト横断リサーチ

指定された Romptn 記事21本を、Yoitomoshi Art Generator の Prompt Library、Prompt Helper、 履歴・補完、投稿準備にどう取り込めるかという観点で横断整理したレポート。 記事本文は辞書資産として有用だが、アプリ実装では「大量のタグ表」ではなく、制作目的別の短い操作導線へ変換するのが効果的。

調査日: 2026-05-16 対象: Romptn 21記事 + 関連公式GitHub 2件 納品: 単一HTML 対象アプリ: Yoitomoshi Art Generator

結論

1. 最優先は Prompt Library の体系化

表情、視線、髪、年齢、肌、メイク、靴、構図、背景、照明、色、画風、品質、negative まで、記事群はタグの網羅性が高い。既存の resources/prompt-library.yoitomoshi.ja.yaml を、制作目的別のカテゴリとタグチップに拡張する価値が最も高い。

実装優先度 A 低リスク

2. 拡張機能は導入よりUX抽出

Prompt All in One は公式GitHubでもメンテナンス停止が明記されているため、Forgeへそのまま入れる対象ではなく、翻訳、重み調整、並べ替え、無効化、履歴のUI発想だけをReact側へ取り込むのが妥当。

直接導入は非推奨 UI参考

3. 投稿準備はアプリの出口機能にできる

投稿サイト記事は、生成後の「AI生成明記」「規約確認」「タグ・キャプション作成」「作風と投稿先の相性確認」を示している。Historyから投稿準備チェックリストを出すと、生成後ワークフローの価値が上がる。

実装優先度 B 出口導線

推奨方針

「Romptnの記事をそのままコピーした辞書」ではなく、ローカル制作補助として再構成する。 具体的には、Prompt Helper が自然文や現在のpromptから不足カテゴリを検出し、短い候補セットを提示する。 ユーザーが必要なタグだけをチップで追加し、重み・順番・positive/negative移動を既存UIで調整できる形がよい。

記事別メモ

記事 要点 Yoitomoshi向けの解釈
表情・顔
2026-04-01		 喜怒哀楽、恐怖、驚きなどの表情タグと、Inpaint / ControlNet reference_only / ADetailer による表情差分。 「表情差分」レシピを作り、seed固定、顔だけInpaint、ADetailer顔補修を一括で準備する導線が有効。
目線・視線
2026-04-01		 カメラ目線、よそ見、見上げ、見下ろし、横目、振り返りなどの視線タグ。 Prompt Helperで「こちらを見る」「横を見る」など自然文から視線タグへ変換するルールを増やす。
髪型・髪色・髪質・髪飾り
2026-04-01		 髪の長さ、女性/男性髪型、前髪、質感、色、髪飾りなど約200種。髪だけ変える場合はInpaintを推奨。 髪カテゴリはタグ数が多くなりやすいので、長さ、前髪、結び方、色、アクセサリに分けて絞り込みUIを用意する。
靴・靴下
2026-04-01		 靴はスニーカー、ハイヒール、ローファー、ブーツなど。靴下は丈別の指定。 全身絵の「足元が曖昧」対策として、衣装カテゴリ内の小カテゴリへ追加。足元を出す構図との組み合わせが必要。
ポーズ・構図
2026-04-01		 プロンプト、ControlNet、OpenPose Editorでポーズ・構図を制御。全身、手、腕、足、座り、寝姿勢など。 ControlNet Builderと接続し、自然文のポーズ指定からOpenPose候補、縦横比、先頭配置を提案する。
年齢指定
2026-04-01		 年齢は数値だけでは反映が弱く、年齢層タグや外見特徴の補正が必要。Age Slider LoRAも万能ではない。 年齢はセンシティブな属性なので、通常タグ辞書には入れても自動推奨は控えめにし、成人/高齢など安全な範囲の表現に寄せる。
肌色・肌質
2026-04-01		 肌色は「skin」形式、肌質は光沢・油分・汗・水滴など。モデルによって効きが変わる。 肌タグは色より質感を優先して候補化する。色指定はモデル依存・表現配慮が必要なため、明示操作時のみ追加する。
化粧・メイク
2026-04-01		 顔全体、眉、目元、頬、唇の5カテゴリ。自然メイク、濃いメイク、アイライン、リップなど。 表情・顔カテゴリと統合し、キャラ印象を変える「メイク」サブカテゴリを作る。ADetailerとの併用にも向く。
背景・風景・場所
2026-04-01		 天気、自然、日常、公共施設、エンタメ、国/地域など。背景メインは横長、wide shot、no humans が有効。 背景だけ生成、人物あり、素材化の3用途でプリセットを分ける。5W1H式の質問UIにすると初心者が選びやすい。
光・ライティング
2026-04-01		 自然光、方向、光の効果、強さ、明るさ調整。モデルによって明るさ・彩度・コントラストの効きが変わる。 照明は「明るくする」より「光源方向・時間帯・質感」で提案する。Preflightで背景/表情/構図との不足を見せる。
品質
2026-04-01		 masterpiece、ultra detailed などの品質タグ、negative、ADetailerとの組み合わせ。 品質タグを足しすぎるとPromptが膨らむため、QuickPresetは「軽量」「標準」「高品質」「顔補修」の4段階程度にする。
色
2026-04-01		 118種類の色指定。赤、黄、緑、青、紫、白、黒、グレーなど系統別。 色は髪、目、服、背景へ対象を明示して挿入する必要がある。単独タグではなく「対象 + 色」のチップ化が向く。
画風
2026-04-01		 画風変更はプロンプト、ControlNet LineArt、LoRA、Hypernetworks。LoRA等は固定力が高い。 画風はPromptだけで完結させず、LoRA選択・ControlNet LineArt候補と一緒に出す。Forge拡張のGradio埋め込みは不要。
リアル・実写系
2026-04-01		 リアル系はモデル選択が重要。品質タグ、img2img、Interrogate CLIPの活用も紹介。 モデル種別がrealisticの時だけ、RAW photo系、光源、レンズ、肌質、negativeの推奨セットを出す。

実装候補の比較

候補 根本対応 実装コスト 評価
A Prompt Library拡張 高い 低〜中 既存YAMLとPrompt Library UIに沿う。記事のタグ資産を一番安全に活用でき、外部依存も増えない。
A Prompt Helperの目的別候補 高い 中 自然文から「表情」「髪」「背景」「照明」「negative不足」を候補化できる。初心者向け価値が大きい。
B ローカルタグ補完 中〜高 中 TagCompleteの発想をReact側へ移植。辞書、History、LoRA trigger、favoritesを候補ソースにできる。
B 投稿準備カード 中 低〜中 Historyから公開前チェックを作るだけなら安全。各投稿サイトAPIや自動投稿は規約・認証リスクがあるため後回し。
C Prompt All in One導入 低 中〜高 メンテナンス停止、Gradio UI依存、Forge/API専用運用との相性が悪い。直接導入は採用しない。

機能へ落とす場合の設計案

Phase 1

Prompt Library: Romptn inspired packs

タグを以下のように分けて追加する。既存カテゴリへ無理に押し込まず、検索・フィルタしやすい粒度を維持する。

人物: 表情 / 視線 / 髪 / メイク / 肌質 構図: 立ち絵 / 座り / 手足 / カメラ距離 / 余白 環境: 背景 / 天気 / 時間帯 / 場所 / 季節 仕上げ: 品質 / 照明 / 色調 / 画風 / negative
Phase 2

Prompt Helper: 不足診断と短い候補

現在のpromptを解析し、足りないカテゴリを提示する。長大な辞書一覧を出すより、「今の画像で効く候補」を3〜8個に絞る。

例: 表情が未指定です -> smile / serious / surprised 背景が弱いです -> city street / forest / classroom 全身構図です -> shoes + socks + full body negative
Phase 3

Tag Autocomplete風のローカル補完

入力中にPrompt Library、History review、LoRA trigger、QuickPresetから候補を出す。NSFW語を含む外部booru辞書は最初から混ぜず、ローカル管理の安全な辞書を基準にする。

Phase 4

投稿準備チェックリスト

History画像に対して、投稿サイト選定、AI生成明記、モデル/LoRAライセンス確認、タイトル、キャプション、タグ候補を出す。自動投稿ではなく手元の整理に留める。

注意点

記事のプロンプトはモデル依存

年齢、肌色、明るさ、彩度、リアル/アニメ表現は、モデル、LoRA、seed、解像度で効き方が変わる。アプリでは「必ず効く」と表示せず、モデル依存の候補として扱う。

Prompt All in Oneは直接導入しない

公式GitHubでメンテナンス停止が明記されている。ForgeのAPI専用起動方針とも衝突しやすいため、UXだけをReact側で再実装する。

NSFW/投稿規約の扱い

TagComplete公式はデフォルトタグリストにNSFW語が含まれると注意している。公開・投稿導線ではAI生成明記、成人向け規約、モデルライセンス、サイトごとの禁止事項を確認させる。

辞書追加時の検証

Prompt Library YAMLを触る場合は、既存AGENTS.mdの通り resources/prompt-library.yoitomoshi.ja.yaml を js-yaml でparseし、カテゴリ数、タグ数、追加タグの存在を確認する。

次にやるなら

  1. Prompt Library増補案をJSON/YAML設計に落とす。 まずは表情、視線、髪、背景、照明、negativeの6カテゴリだけを追加対象にする。
  2. Prompt Helperのルールを増やす。 自然文から「表情差分」「髪だけ変更」「背景を綺麗に」「投稿向けタグ」などへ変換する。
  3. PromptTagChipsに一時無効化を足す。 Prompt All in Oneの有用な操作だけを既存React UIへ持ち込む。
  4. Historyの投稿準備カードを作る。 AI生成明記、タグ、キャプション、規約確認のチェックリストをローカルで出す。
