## Remaining Risks

- No broad `--apply` has been run against normal `userdata`.
- The next operational phase should use a small real-data dry-run first:
  1. `npm.cmd run dictionary:enrich:meanings -- --limit 10 --dry-run --json-out output/prompt-dictionary-meanings-dry-run.json`
  2. `npm.cmd run dictionary:enrich:meanings:review -- --from-report output/prompt-dictionary-meanings-dry-run.json --export output/prompt-dictionary-meaning-review.json`
  3. Manually edit accepted entries.
  4. `npm.cmd run dictionary:enrich:meanings:review -- --import output/prompt-dictionary-meaning-review.json`
  5. Rebuild/check runtime dictionary only after review import is correct.
- Provider schemas can still drift. The fixture regression protects local scoring behavior, not every future API payload shape.
- Review UI is intentionally light. It exposes counts and commands, not an in-app editor.
- Repo-wide dirty state includes other Prompt Daijiten/source-expansion work beyond this reliability slice. This closeout does not claim the whole repo is release-clean.
